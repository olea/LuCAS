3.3. Requisitos Funcionales de la Aplicación

3.3.1. Herramientas de administración

La administración del sistema deberá asegurar un funcionamiento coherente del sistema. Los principales elementos a considerar son:

  • Creación de eventos programados para la ejecución de tareas periódicas

  • Control de arranques, congelación o paradas de servicios en el sistema

  • Gestión de los usuarios

  • Gestión de los spool de impresión, workflows, batch

  • Gestión de las conexiones (sesiones, conectores, llamadas externas, ...)

  • Generación de trazas de ejecución de los procesos (log)

3.3.2. Copias de seguridad

El sistema tiene que estar protegido contra cualquier eventualidad y ser restaurado completamente. Para ello es necesario contemplar una correcta salvaguardia del sistema mediante la realización de copias de seguridad. Estas pueden ser más o menos efectivas en función de lo que ofrezca el propio fabricante:

  • Herramienta propia para la restauración del sistema

  • Solución complementaria o de tercero

  • Mediante copia simple de base de datos

3.3.3. Archivado

Cuando el sistema adquiera cierto volumen de información en base de datos o sistemas de ficheros se deberá poder realizar descargas o archivado progresivo de los datos históricos. Algunos productos tiene previsto esta problemática y disponen de conectores para software de tercero que facilitan descargar la base de datos y también individualmente las tablas que acaban teniendo mucha carga de registros.

3.3.4. Monitorización del sistema

El ERP debe incluir un control de monitorización del rendimiento o la carga del sistema. Esto debe permitir detectar bloqueos, necesidades de actualización de la configuración de los recursos lógicos o físicos del sistema.

3.3.5. Exportación/ Importación de datos

Disponer de herramientas para la exportación o importación de datos en el sistema que permitan facilitar todo tipo de actividades de cargas en particular la propia implantación.

3.3.6. Correo electrónico / Fax

Además de poder imprimir en papel los documentos como las facturas, informes, albaranes, ... el sistema debe ofrecer la posibilidad de enviar esos documentos por fax o por correo electrónico.

Deberá integrarse con el correo Postfix SMTP / Lotus Notes Domino Servidor corporativo de la entidad empresarial, indicar la certificación de integración de la solución.

Indicar también las necesidades de servidor de fax que serían ser soportados (no incluir el software en la oferta). la entidad empresarial dispone de IMPRESORAS / FAX modelo LANIER 5622.

3.3.7. Ofimática

Las herramientas ofimáticas deberán permitir a los usuarios poner en forma rápidamente informes, tablas, gráficos, ... con datos exportados hacia las mismas desde el sistema.

3.3.8. Workflow

Un sistema de Workflow permitirá automatizar y controlar la ejecución de ciertos procesos dónde tienen que intervenir en cadena de distintos participantes. Existen diferentes casos dónde una herramienta de este tipo se hace indispensable para asegurar el correcto cumplimiento de plazos de ejecución o calidad en los procesos.

Se recomienda siempre considerar una herramienta del fabricante ERP aunque también pueda enviar/ recibir mensajes hacia otros sistemas o correo electrónico lo que permitan externalizar el Workflow.

3.3.9. Gestión Documental

La entidad empresarial prevé integración de un sistema de gestión documental con el sistema integrado de gestión para que este pueda disponer de un acceso a documentos electrónicos completando así la cobertura de los procesos de negocio. El adjudicatario deberá indicar la capacidad de integración del sistema para poder ligar documentos al ERP. Se valorará tanto la oferta de soluciones de terceros pero teniendo en cuanta la facilidad de integración de las mismas.

3.3.10. Reporting

La realización o personalización de informes a medida para las necesidades del negocio es necesaria. El producto ERP debe permitir tanto la realización de informes a medida partiendo de los datos del sistema y poder ampliar también modelos existentes en el sistema estándar. También se valorará las capacidades de configuración que puede realizar el mismo usuario final sin asistencia por ejemplo de listados dinámicos.

  • Desarrollo de informes a medida

  • Personalización de los informes estándar

  • Configuración de listados a nivel de usuario final
