3.12. Requisitos de mantenibilidad

Entorno de desarrollo

La solución debe disponer de un entorno de desarrollo propio con el que se construye el núcleo del sistema que deberá estar basado en una estructura de programación 4GL aunque en muchos casos esta es propia a cada fabricante.

Entornos

El sistema ERP debe separar los diferentes entornos de sistema pero al menos se deberá tener desarrollo, producción y pruebas. Una vez el ERP este en producción es crítico que este se mantenga en correcto funcionamiento independientemente de los cambios funcionales o técnicos que se quieran desarrollar o probar. Para ello es necesario, que existan mecanismos de mantenimiento que faciliten la administración de estos entornos.

Es obligatorio considerar una arquitectura basada en 3 entornos:

  • Entorno de Desarrollo

  • Entorno de Preproducción

  • Entorno de Producción
