Capítulo 2. Descripción general

2.1. Perspectiva del producto

2.1.1. Interfaces del sistema

En esta sección pondremos el producto que se está analizando bajo la perspectiva de otros productos relacionados. En el caso de ser una ER de un producto pequeño que forma parte de un producto más grande, se indicará el papel que forma el pequeño producto dentro del conjunto, así como las interfaces con el resto de piezas. (Se puede redirigir a otro documento). En caso de que la ER sea de un sistema completo (sin subpartes), se indicará de esa manera.

2.1.2. Interfaces de usuario

En esta sección se pondrá el producto bajo el punto de vista de su usuario. Se indicarán las interfaces y formularios que sean de obligada implementación y que estén perfectamente definidos (como, por ejemplo, en programas que sirvan para cumplimentar impuestos habrá que indicar el formato de los impresos...)

2.1.3. Interfaces hardware

La solución propuesta deberá poder apoyarse en la infraestructura de redes LAN -TCP/ IP actualmente utilizadas por la entidad empresarial. la entidad empresarial plantea la creación de una VPN que será accesible mediante MODEM RTC 56 Kb. El sistema propuesto deberá demostrar poder tener acceso en esta modalidad tanto para los usuarios operacionales como para los administradores.

2.1.4. Interfaces software

En esta sección se indicarán aquellos productos software cuyo uso se considere requisito inprescindible, indicando nombre y versión de los mismos, y el motivo de considerar esa dependencia como requisito.
