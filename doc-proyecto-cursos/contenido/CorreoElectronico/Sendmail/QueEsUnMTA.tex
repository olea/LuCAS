\porcion{?'Qué es un MTA?}

\traductor{}

Las siglas MTA significan <<\emph{Mail Transfer Agent}>> que una
traducción aceptable puede ser \emph{Agente de Transferencia de
Correo}. Esta definición se contrasta con MUA, que significan
<<\emph{Mail User Agent}>> o podríamos llamarlo \emph{Agente de
Correo del Usuario}.  Esta distinción se realiza para dividir la tarea
de envío de correo, que están encargados los MTAs, de la tarea de
redacción y edición de correos, realizados mediante un MUA.

El protocolo\footnote{Los protocolos para las computadoras son como
los idiomas para los humanos.} más común utilizado entre los MUA y los
MTAs de Internet es SMTP o \emph{Simple Mail Transfer Protocol} en inglés
o \emph{Protocolo de Transferencia de Correo Simple}.
El servidor \comando{sendmail} es un MTA, o también denominado 
\emph{servidor SMTP}. 

Una transacción posible al enviar un email entre el MUA y el MTA
se describe a continuación:

\begin{itemize}

\item El MUA se conecta con el MTA.

\item El MUA envia el remitente del correo.

\item El MTA acepta el remitente como un remitente válido.

\item El MUA envia el destinatario del correo.

\item El MTA acepta el destinatario como válido.

\item El MUA envia el ``subject'' o ``tema'' al MTA y el cuerpo del mensaje

\item El MTA lo acepta

\item El MUA termina la conexión

\end{itemize}


Los MTA pueden conectarse entre ellos para realizar la entrega de mail
utilizando también el protocolo SMTP. Inclusive pueden convertir correos
de varios formatos si es necesario.
