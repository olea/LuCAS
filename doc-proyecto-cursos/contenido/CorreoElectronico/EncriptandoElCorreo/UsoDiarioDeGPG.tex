Una vez configurado el programa de cifrado, lo primero que se debe
hacer es \emph{generar un certificado de revocación}, el cual servirá
para poder anular el par de claves en caso de olvido de la contraseña,
o robo de la clave privada. Esto se hace de la siguiente forma:

\begin{vscreen}
usuario@maquina:~$ gpg --output cert-revocacion.asc --gen-revoke Usuario

sec  1024D/62B70584 2001-04-22   Usuario (233-8847) <usuario@maquina.dominio.com>

Create a revocation certificate for this key? y
Please select the reason for the revocation:
  1 = Key has been compromised
  2 = Key is superseded
  3 = Key is no longer used
  0 = Cancel
(Probably you want to select 1 here)
Your decision?
\end{vscreen}
%$

El argumento que se le debe pasar al parámetro \comando{--gen-revoke}
es el nombre que previamente se ingresó cuando se configuró GPG. El
programa pide la razón por la cual se genera el certificado de
revocación y un comentario, luego GPG genera el archivo
\archivo{cert-revocacion.asc} (que es un archivo de texto) con el
certificado incluido. Este archivo debería guardarse en algún medio de
almacenamiento y esconderse donde nadie tenga acceso. Una de las
opciones válidas puede ser imprimir el archivo, guardar la hoja en
algún lugar seguro y luego borrar el archivo.

Por cada persona a la que se quiera enviar mensajes cifrados, se debe
tener almacenada su clave pública en el \emph{anillo de llaves} de
GPG. Para ver la lista de claves públicas que se tiene en cualquier
instante, se ejecutará el siguiente comando:

\begin{vscreen}
usuario@maquina:~$ gpg --list-keys
/home/usuario/.gnupg/pubring.gpg
--------------------------------
pub  1024D/62B70584 2001-04-22 Usuario (233-8847) <usuario@maquina.dominio.com>
sub  2048g/7459EB6A 2001-04-22
\end{vscreen}
%$

Inicialmente se tendrá únicamente la propia clave pública en el
anillo. Cuando se reciba alguna clave pública de otra persona, se la
debe agregar al anillo de la siguiente manera:

\begin{vscreen}
usuario@maquina:~$ gpg --import <nombre_archivo>
\end{vscreen}
%$

Donde \comando{nombre\_archivo} corresponde al archivo donde se
tengan la o las claves públicas a ingresar.

De manera similar, se necesitará exportar la propia clave pública a un
archivo de texto para poder enviarla a aquellas personas que quieran
enviar mensajes cifrados. Esto se hace como sigue:

\begin{vscreen}
usuario@maquina:~$ gpg --armor --output <archivo_exportado> --export <nombre>
\end{vscreen}
%$

Donde \comando{archivo\_exportado} será el archivo donde se
almacenará la clave pública en formato de texto, y \comando{nombre}
deberá corresponder con el nombre o la dirección de correo electrónico
propia que se haya ingresado en los datos del programa. De hecho, este
comando sirve para exportar cualquier clave pública, la propia o la de
cualquiera que esté en el anillo, proveyendo el correspondiente nombre
o dirección electrónica.
