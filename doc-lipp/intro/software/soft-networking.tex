\subsection{Introducción a las redes}

¿Desea comunicarse con el mundo? {\linux} soporta dos protocolos 
básicos en UNIX: TCP/IP y UUCP. El protocolo TCP/IP 
(Transmission Control Protocol/Internet Protocol) es el paradigma de
redes que permite a los sistemas de todo el mundo intercomunicarse
mediante una sola red, {\bf Internet.} Con {\linux}, TCP/IP y una conexión
a Internet \NT{Que en España han bajado mucho de precio o son gratuitas},
puedes comunicarte con usuarios y ordenadores a través del correo
electrónico, noticias, y transferencia de ficheros por FTP.

La mayor parte de las redes TCP/IP utilizan Ethernet como transporte físico
de la red. {\linux} soporta las tarjetas de red Ethernet más usuales y los
interfazs para ordenadores personales, como los adaptadores pocket Ethernet 
y PCMCIA.

Sin embargo, y debido a que no todo el mundo tiene una conexión Ethernet en
casa, {\linux} soporta también {\bf SLIP} (Serial Line Internet Protocol, protocolo
de Internet por línea serie) y {\bf PPP} (Point-to-Point Protocol, protocolo de
punto a punto), que proporcionan el acceso a Internet a través de un módem.
Muchos negocios y universidades disponen de servidores PPP a los que conectarse.
De hecho, si su sistema {\linux} dispone de una conexión Ethernet y un módem,
puede convertirse en un servidor SLIP o PPP para otros ordenadores.

NFS (Network File System, sistema de ficheros de red) le permite compartir ficheros
con otras máquinas de la red de forma transparente. FTP (File Transfer
Protocol, protocolo de transferencia de ficheros) le permite transferir ficheros
desde y hacia otro ordenador. {\tt Sendmail} envía y recibe correo electrónico
a través del protocolo SMTP. C-News e INN son sistemas de noticias basados en
el protocolo NNTP; y {\tt telnet}, {\tt rlogin}, {\tt rsh} y {\tt ssh} le permiten
autentificarse y ejecutar órdenes en otras máquinas de la red. Con {\tt finger}
 podra obtener información sobre otros usuarios de Internet.

{\linux} también soporta la conectividad con Microsoft Windows gracias a
Samba\footnote{ Véase {\em Samba: Integrating UNIX and Windows}, Copyright
1998 Specializad System Consultants.} y conectividad con Macintosh
a través de AppleTalk y LocalTalk.  También se incluye el soporte
para el protocolo IPX de Novell.

{\linux} tiene disponible una batería completa de lectores de correo y noticias,
entre ellos {\tt elm}, {\tt pine}, {\tt mutt}, {\tt rn}, {\tt nn}, y {\tt
tin}. Cualquiera que sea su preferencia, puede configurar el sistema {\linux}
para que envíe y reciba noticias y correo electrónico desde todo el mundo.

El sistema proporciona un interfaz estándar de programación de sockets 
UNIX. Se puede migrar a {\linux} prácticamente cualquier programa que use
TCP/IP. El servidor de las X para {\linux} también soporta TCP/IP, y las
aplicaciones que se ejecutan en otros sistemas pueden usar la pantalla
de su sistema local para visualizarse.

En el Capítulo~\ref{chap-networking} trataremos de la instalación del
software TCP/IP, y con él SLIP y PPP.

UUCP (UNIX-to-UNIX Copy, copia de UNIX a UNIX) es un mecanismo ya bastante
antiguo para transferir correo electrónico y noticias entre máquinas UNIX.
Históricamente, las máquinas UUCP estaban conectadas por líneas telefónicas
a través de un módem, pero UUCP es capaz igualmente de transferir datos 
a través de una red TCP/IP. Si no dispone de acceso a una red TCP/IP
(acceso fácil de obtener en España con el acceso gratuito a Internet),
puede configurar su sistema para enviar ficheros y correo electrónico 
utilizando UUCP. Para más información, véase el capítulo~\ref{chap-networking}.
