<h2>Una arquitectura para las fases de reproducción y publicación<p>
<center><a href="herramientas-reproduccion-publicacion.png"><img src="herramientas-reproduccion-publicacion.png" alt="Arquitectura para reproducción y publicación" border=0"></a></center>

<h2>Explicación de la arquitectura para las fases de reproducción y publicación (1)<p>
<dl>
<dt>Fuentes del documento
<dd>Son las fuentes del documento.<br> Se prefieren en un
              formato estructurado como DocBook-XML,<br>válido para cualquie formato     «homologable». <br>Se aloja en el cvs.
<dt>Instrucciones de reproducción
<dd>Fichero con las instrucciones que debe seguir la imprenta-e para crear los formatos de reproducción
<dt>Servidor CVS
<dd>Herramienta de trabajo en grupo para el desarrollo de documentación y programas.
<dt>Registro de documentos
<dd>Registro de los documentos a publicar, con URL referencia a sus fuentes.
<dt>OMF.xml
<dd>Ficha bibliográfica del documento expresada con metadatos OMF.
<dt>imprenta-e
<dd>La herramienta que gestiona todo el proceso de publicación.
<dt>bibliotecario-e
<dd>Herramienta de registro automático de un documento en el acervo bibliotecario.
<dt>impresor-e
<dd>Herramienta automática de creación de los formatos de reproducción.
<dt>Servidor DAV
<dd>Herramienta de trabajo en grupo para compartición de ficheros.
</dl>

<h2>Explicación de la arquitectura para las fases de reproducción y publicación (2)<p>
<dl>
<dt>PDF
<dd>Documentos en formato PDF
<dt>HTML
<dd>Documentos en formato HTML
<dt>HTML.tgz
<dd>Exactamente el mismo documento de arriba pero empaquetado con tar+gz para su copia y distribución.
<dt>src.tgz
<dd>Código fuente del documento empaquetado con tar+gz para su copia y redistribución.
<dt>Servidor web
<dd>¿Alguien necesita que se lo explique?
<dt>IU Consultas
<dd>Interfaz de usuario para las consultas en la web (web dinámico).
<dt>publicador-e
<dd>Herramienta para incluir en un web estático las referencias a los documentos.
<dt>Web estático
<dd>Árbol estático de páginas web para publicar los documentos.
<dt>Navegador web
<dd>Tecnología secreta para conseguir la fusión fría.

</dl>



<h2>Propuesta del interfaz de usuario<p>
<center><a href="web-lucas-v4.png"><img src="web-lucas-v4.png" alt="Propuesta portada del IU" border=0"></a></center>


<h2>Explicación de la propuesta de interfaz de usuario<p>

<dl>
<dt>Cadena de búsqueda
<dd>Widget para introducir los términos de una búsqueda de documentos.
<dt>Área de búsqueda al estilo yahoo
<dd>Vistas del árbol de documentos restringidas con los términos recogidos en el widget citado.
<dt>Enlace a la forja de docs
<dd>La forja de documentos es el ambiente cooperativo de desarrollo de los mismos.<br>El objetivo es implantarlo con una herramienta tipo SourceForge.
<dt>Enlace al sistema de publicación TLDP-ES
<dd>El sistema de publicación es la herramienta para el seguimiento del control de calidad del documento a lo largo del ciclo de edición.
<dt>Directorio de recursos y herramientas para el autor.
<dd>Este directorio servirá para poner en manos del autor todo el taller de trabajo, en forma de herramientas y métodos, para desarrollar su trabajo dentro de LuCAS v4.
<dd>Novedades docs
<dt>Anuncios de publicación de documentos en TLDP-ES
<dd>Eventos, congresos, campañas
<dt>Tablón de anuncios y convocatorias de interés para la comunidad de usuarios y desarrolladores TLDP-ES
<dd>Novedades comunidad doc libre
<dt>Anuncios sobre la actividad de proyectos de documentación libre.
<dd>Otros enlaces secundarios 
<dt>Enlaces de contacto, derechos, etc.
</dl>
