Iniciando el proceso principal (postmaster)

Nada puede ocurrirle a una base de datos sino se está ejecutando postmaster. Como administrador del sistema, existe un cierto número de cosas que debería recordar antes de iniciar postmaster. Esto se discute en las secciones de instalación y configuración de este manual. Sin embargo, si usted ha instalado Postgres siguiendo exactamente las instrucciones tal como se indica, el siguiente comando debería ser suficiente para poder iniciar postmaster. 

% postmaster

En algunas ocasiones, postmaster exhibe ciertos mensajes que pueden ser de utilidad para la resolución de problemas. Si desea ver los mensajes de depuración que presenta postmaster , puede iniciarlo con la opción -d y redirigir la salida a un archivo de registro: 

% postmaster -d > pm.log 2>&1 &
Si no desea ver estos mensajes, puede teclear lo siguiente 
% postmaster -S
y, de este modo, postmaster trabajará en modo "S"ilencioso. Note que no hay un ampersand ("&") al final del último ejemplo, por lo que postmaster estará ejcutándose en primer plano.
