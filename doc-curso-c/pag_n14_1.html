14.1.- Las páginas man                                                   



Cuando estamos usando un sistema como Linux a menudo nos surgen preguntas acerca de un determinado comando propio de UNIX. Los creadores de UNIX, lanzaron un sistema de ayuda en linea muy eficiente y que nosotros llamaremos 'páginas man'. Para encontrar información acerca de un determinado comando UNIX tendríamos varias opciones: Comprar un libro referente a UNIX-Linux, llamar a un amigo, preguntar al profesor o usar las páginas man. Las páginas man son archivos comprimidos de texto que contienen información acerca de todos los comandos UNIX, funciones propias del sistema operativo y funciones de lenguaje C. Para encontrar información acerca de un determinado comando UNIX lo haremos de la siguiente manera:

man nombre_comando


Las páginas man son archivos de texto ordenadas en secciones que abarcan los siguientes temas:

Sección 1 Comandos de Unix

Comandos que se usan desde una shell.


Sección 2 Llamadas al sistema

Funciones propias del sistema Linux.


Sección 3 Funciones de C

Descripción de la mayoria de las funciones del lenguaje C


Sección 4 Archivos especiales

Archivos que se encuentran en el directorio /dev


Sección 5 Formatos de archivo

Sección 6 Juegos

Sección 7 Otros paquetes sueltos interesantes

Protocolos de red, codigos ASCII, etc


Sección 8 Comandos para la administración del sistema

Comandos que sólo puede ejecutar el root o administrador del sistema


Sección 9 Rutinas del núcleo Linux

Sección obsoleta



Como estamos aprendiendo a programar en C, en un principio la sección que nos interesa es la número 3. El inconveniente de esta ayuda en linea es el idioma, debido a que las paginas están escritas en inglés. Afortunadamente muchos y buenos programadores y traductores han traducido al español la mayoría de ellas. Puede que nuestro sistema tenga instaladas por defecto las páginas traducidas al español, pero tambien puede que a pesar de estar instaladas no seamos capaces de leerlas cuando intentamos leer algún tipo de ayuda. Para ello ejecutaremos el siguiente comando:

# echo $MANPATH


Este comando arrojará por pantalla algo parecido a esto:

/usr/local/man:/usr/man:/usr/X11R6/man:/usr/openwin/man:/usr/man/allman:/usr/man/de:
/usr/man/es


Ocurre que cuando intentamos localizar algún término mediante el comando man, el programa comenzará a buscar en el primer direcorio que aparece '/usr/local/man/'. Si el término buscado no se encontrara, buscará en el siguiente directorio, '/usr/man/' y así has ta terminar el listado de directorios separados por ':'. Si nos fijamos en el listado ofrecido por 'echo $MANPATH', vemos que el directorio '/usr/man/es' está al final del listado y es precisamente ese directorio el que contine la mayoría de la paginas man en español. Desafortunadamente está al final de la lista con lo que es casi inposible que el comando man nos de algun tipo de respuesta en español. La solución es situar el directorio '/usr/man/es' al principio del listado. Esto lo puedes solucionar de las siguientes maneras:

1.- Ejecutando la siguiente linea:

MANPATH="/usr/man/es:/usr/local/man:/usr/man:/usr/X11R6/man:
/usr/openwin/man:/usr/man/allman:/usr/man/de"


Hemos situado el direcorio /usr/man/es al principio de la lista, con lo que conseguieremos que el comando man comience su búsqueda en el directorio insertado en primer lugar, es decir, primero buscará en español y despues se dirigirá a los idiomas restantes. En todo caso esta es una solución temporal, pues cuando eliminemos la shell o cambiemos de terminal esta trayectoria será ignorada y volveremos a tener las páginas man en inglés. Veamos a continuación otra forma de conseguir permanentemente las págians man en español.

2.- En nuestro directorio de trabajo '/home/nombre_de_usario' o en el directorio '/root' si salimos copmo root, buscaremos y editaremos el archivo '.profile' y en este archivo escribiremos las siguientes lineas:

MANPATH="/usr/man/es:/usr/local/man:/usr/man:/usr/X11R6/man:/usr/openwin/man:
/usr/man/allman:/usr/man/de"

export MANPATH


Con estas lineas conseguiremos que cada vez que arranquemos nuestro sistema la trayectoria para el comando man será interpretada en todas las consolas y shells.

3.- Una tercera solución sería escribir el comando man como sigue:

man -M /usr/man/es <nombre del término buscado>


Con lo que el comando man sólo buscará en el directorio referenciado, ignorando los demás.

Podemos obtener una simple descripciónde un comando o función de una sola línea ejecutando cualquiera de los siguientes comandos:

man -f <nombre_de_comando>
whatis <nombre_de_comando>


Por ejemplo si ejecutamos algunois de los dos anteriores comandos sobre fprintf, obtendremos el siguiente resultado:


fprintf (3) - formatted output conversion


Donde el número entre paréntesis es el número de sección en el que se encuentra el comando o función buscada. Como nuestro objetivo es aprender la programación en C cuando solicitemos ayuda con man para una determinada función usaremos man de la siguiente manera:

man 3 nombre_de_funcion


Donde el número 3 indica a man que busque sólo en la sección 3. Esto es debido a que hay nombres de comandos y funciones con idéntico nombre y man se va por defecto a la sección 1. Veamos que ocurre cuando solicitamos informacion sobre scanf() mediante 'man 3 scanf'. A continuación el comienzo de la página man para scanf():



SCANF(3) Linux Programmer's Manual SCANF(3)



NAME

scanf, fscanf, sscanf, vscanf, vsscanf, vfscanf - input

format conversion


SYNOPSIS

include <stdio.h>

int scanf( const char *format);

int fscanf( FILE *stream, const char *format . . .);

int sscanf( const char *str, const char *format, .....);


include <stdarg.h>

int vscanf( const charr *format, va_list ap);

int vsscanf( const char *str, const char *format, va_list ap);

int vfscanf( FILE *stream, const char *format, va_list ap);


DESCRIPTION

The scanf family of functions scans input according to a

format as described below. This format may contain conversion

specifiers; the results from such conversions, if

any, are stored through the pointer arguments. The scanf

function reads input from the standard input stream stdin,

fscanf reads input from the stream pointer stream, and

sscanf reads its input from the character string pointed

to by str.


The vfscanf function is analogous to vfprintf(3) and reads

input from the stream pointer stream using a variable

argument list of pointers (see stdarg(3). The vscanf

function scans a variable argument list from the standard

input and the vsscanf function scans it from a string;

these are analogous to the vprintf and vsprintf functions

respectively.


En el caso de que el comando o función buscada no se encuentre disponible el mensaje de Linux será:

No manual entry for .....
