10.18.- Detección del fin de un fichero con feof()


Podemos usar las funciones de apertura, escritura y lectura de ficheros directamente, sin comprobar que todo ha ido correctamente y lo más seguro es que todo marche correctamente, pero existes múltiples factores que pueden anular la operación que queremos hacer con ficheros. Nosotros hemos usado los valores devueltos por las diferentes funciones para detectar algún posible error. Aparte de nuestro método existen otros cauces para detectar errores.

En stdio.h aparece definida una función externa que devuelve un valor de tipo int. Hablamos de la función ferror():


#include <stdio.h>

int ferror(FILE *stream);

Devuelve 0 si todo ha ido correctamente.



Imagina un programa que use la función ferror() más de una vez. Si en el primer uso de ferror() se detectara algún error en la operación con ficheros, antes de usar el segundo ferror(), se debe de usar la función clearerr() sobre el fichero apuntado. Cuando se abre un fichero al mismo tiempo se activa un indicador de error para ese fichero con valor 0. Si ocurre algún error en la lectura-escritura del fichero abierto, el indicador de error se torna distinto de 0. Antes de volver a usar ferror(), debemos de situar el indicador de error a 0 mediante claearerr(), en caso contrario, siempre detectaría error la función ferror(). Es una buena costumbre incluir la función clearerr() antes del uso de ferror(). En todo caso esta función no nos permite saltarnos errores de escritura, es decir si intentamos escribir datos en un fichero que no ha podido ser abierto el sistema operativo emitirá por consila un mensaje similar a este:


Segmentation fault


Este mensaje se emitirá independientemente de si se ha ejecutado ferror() o no. En la inmensa mayoría de los casos es muy necesario salir del programa cuando se ha producido un error en el manejo de archivos, por eso recomendamos desde estas lineas salir del programa con exit().


En todo caso podemos resolver tambien los errores con los valores devueltos por las diferentes funciones de manipulación de archivos que tambien dan muy buen resultado y tal vez sean más intuitivas.





Cuando creamos un fichero, Linux añade automáticamnete el indicador de fin de fichero, indicador que se va desplazando hacia el final del fichero a medida que se vamos añadiendo datos. Cuando abrimos un fichero, un varible indicadora del fin de fichero tambien se inicia con valor 0. Cuando intentamos leer datos del fichero más allá de la indicación de fin de fichero, la varible indicador a de fin de fichero adquiere un valor distinto de 0. La función feof() lee la esta varible indicadora de fin de fichero.


Sinopsis feof()

#include <stdio.h>
int feof(FILE *stream)

Devuelve un valor distinto de 0 si intenta leer más allá de la marca EOF del fichero apuntado por stream



Un ejemplo del uso de feof() lo tenemos en la función ConsultarDatos() anteriormente vista. Observa que hemos usado la función de manera que si el indicador de fin de fichero está activo (linea 54) emita un mensaje por consola y luego desactivamos el indicador de error:


54

              if(feof(stream))

55

                   {

56

                    printf("Esa referencia no existe\n");

57

                    clearerr(stream);

58

                   }


Esta función nos puede servir para leer datos de ficheros:



while(!feof(stream))

{


//..... leer datos de fichero


}


Para desactivar el indicador de fin de fichero podemos usar la función rewind() o clearerr().
