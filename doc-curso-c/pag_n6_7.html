6.7.- system()                                              



La funcion system() hace que el sistema operativo ejecute la orden contenida en el interior de los paréntesis. Simplemente pasa una cadena de caracteres al lector de órdenes del sistema operativo y este la ejecuta si puede.  La sintaxis es:
 

#include <stdlib.h>

int system ("cadena de caracteres");


El valor devuelto por system() es 0 si la orden se ejecuta satisfactoriamente.  Según las páginas man de mi sistema Linux , devolvería 127 si la orden es interpretada pero no ejcutada, sin embargo me devuelve 256. Cuando no conoce la orden y por tanto no llega a interpretar dicha orden me devuelve el valor 32512.  Lo que si es seguro es que en todos los sistemas devuelve 0 si la orden es aceptada. Prueba en tu sistema MSDOS y compruébalo, nosotros no lo hemos hecho, pero sí lo hemos desarrllado en Linux. Observa el programa:
 
 

#include<stdio.h>
#include<stdlib.h>

main()
{
int num;

num = system ("mkdir apache"); /*creamos el directorio apache*/
printf("He creado el directorio apache. El valor de num es %d\n", num);

num = system ("mkdir apache");  /*intentamos crear de nuevo el directorio apache*/
printf("He intentado crear de nuevo el directorio apache. El valor de num es %d\n", num);

num = system("rmdir apache"); /*borramos el directorio apache*/
printf("He borrado el directorio apache. El valo de num es %d\n", num);

num = system("mi hermana es torera"); /* intentamos ejecutar una orden rara */
printf ("El valor de num si mi hermana es torera es %d\n", num);
}


La salida del programa sería:
 
 

He creado el directorio apache. El valor de num es 0 

mkdir: cannot make directory 'apache': file exists

He intentado crear de nuevo el directorio apache. El valor de num es 256
He borrado el directorio apache. El valor de num es 0
El valor de num si mi hermana fuera torera es 32512 



mkdir <cadena> y rmdir <cadena> son órdenes del sistema UNIX para crear y borrar directorios respectivamente, pasamos a explicar el programilla.


Observa como funciona el programa y los valores devueltos. En primer lugar nos crea el directorio apache y devuelve el valor 0, ya que ha podido ejecutar la orden.


A continuación vuelve a intentar crear el directorio apache, no va a poder crearlo ya que el directorio ya existe y los sistemas operativos son especialmente cuidadosos en no llamar a distintos directorios de idéntica manera. Entonces el sistema operativo nos responde (en este caso en inglés) explicádonos que no puede crear el directorio apache pues ya existe. En este caso nos devuelve el valor 256(Otros sistemas devuelven el valor 127). El intérprete de órdenes ha podido leer la órden e interpretarla, sin embargo el sistema operativo no la acepta por el motivo comentado.


A continuación borramos nuestro directorio pasándole la orden "rmdir apache" al sistema operativo. la orden la puede leer y además ejecutar pues el directorio apache existe desde hace un par de líneas. En este caso nos ha devuelto un 0. Cuando le intentamos pasar un valor realmente extraño como "mi hermana es torera" (y digo extraño por que ninguno de los que escribimos este manual tenemos hermanas torearas ni tan siquiera hermanas y además este tipo de ordenes no son conocidas) ocurre que no puede interpretar esa cadena de caracteres como una orden para el sistema operativo y devuelve el valor de 32512.
