8.9.- El programa de una compraventa de vehículos

Una antigua empresa productora de leche ha cambiado su enfoque a una compraventa de vehiculos de segunda mano de todo tipo, en particular motos, coches y camiones. El propietario de la compraventa, Mr. Badleg, que no entiende mucho de informática pero ha oido hablar de uniones y estructuras le ha dicho a su sobrino, que si entiende de informática, que ponga todos los vehículos de su negocio en un array de estructuras con uniones anidadas. La compra venta de vehículos se llama madcar y el programa que le hizo su sobrino es madcar.c

Para desarrollar este programa hemos de disponer de una estructura en el que aparezcan diferentes tipos de datos referentes al vehículo, como por ejemplo antigüedad, marca, modelo, potencia, etc, incluso se podría poner una foto del vehículo. Esta estructura distinguirá entre motocicletas, camiones y coches y para distinguier este tipo de datos usaremos una union que contenga tres estructuras que contemplen esta eventualidad. Nosotros opinamos que para empezar será suficiente con los siguientes datos y la siguiente estructura. Si el lector quiere añadir más datos lo dejamos de su cuenta:
 
 
 

struct vehiculo

      {

          char t_vehiculo; //Esto es para el func interno

          int numref; //la referencia del vehículo

          char marca[20];

          char modelo[30];

          int potencia;

          int anyos;

          union 

                   {
                                                            /*No titulamos las estructuras y ya quedan declaradas*/

                       struct

                              {

                               unsigned tara;

                               unsigned p_max;

                               }camion;

                       struct

                               {

                                char sidecar;

                                char pasajero;

                               }moto;

                        struct

                               {

                                char extras[81];

                                char tipo[20];

                               }coche;

                     }clase; //Este es el nombre de la union

           long precio;

};



En esta declaración ocurre que declaramos internamente a vehículo varias variables que si bien algunas son obvias a la definición de vehículo, otras nos servirán para navegar por los diferentes datos del array de estructuras. Una novedad aparece es la no titulación de la union ni de las estructuras contenidas en la union, sin embargo al terminar las mismas sí que las aplicamos un nombre. La razón es que de esta manera ya quedan definidas y listas para introducir datos en ellas, con lo cuál nos ahorramos una definición innecesaria. Si usamos esta notación, no podremos declarar variables de la estructura o union afectada pero sí podremos cambiar los valores contenidos en dichas estructuras o uniones.

La función main() que usaremos tan sólo declarará un array de estructuras del tipo vehículo y de tamaño N y usará dos funciones, una para introducir datos y otra para leerlos:



void main()

        {

          struct vehiculo parque[N];
 
 

          int n;

          n = escribir(parque, N);

          leer(parque, n); 

}


La variable N vendrá definida en el exterior con #define y la variable 'n' recibirá el nº de datos escritos para así poder leer tantos datos como hayan sido escritos. A continuación el desarrollo definitivo del programa:
