11.4.- Modificación de los permisos sobre un archivo, access(), chmod() y fchmod()



Para conocer si un proceso tiene acceso a un determinado archivo u saremos la función access():



Sinopsis de access()

#include <unistd.h>

int access(char *trayectoria, int mode);

Devuelve 0 en caso de acceso posible y -1 caso contrario



El parámetro int mode puede tomar alguno de los valores siguientes:





Opción para int mode

Descripción

F_OK

Solicita una comprobación acerca si el chequeo sobre la existencia de un archivo está permitido - caso que el proceso carezca de permisos de lectura en determinados directorios de char *trayectoria-

R_OK

Comprueba el acceso en modo lectura

W_OK

Comprueba el acceso en modo escritura

X_OK

Comprueba el acceso en modo ejecución


Y los valores que puede tomar la variable errno son:



Código de error

Descripción

EACCES

El accesso solicitado sería denegado, ya sea al fichero mismo o a uno de los directorios en char *trayectoria

EFAULT

char *trayectoria señala fuera del espacio de direciones

accesible.

EINVAL

int mode ha sido especificado incorrectamente.

ENAMETOOLONG

char *trayectoria es demasiado largo

ENOENT

Cierto directorio en char *trayectoria habría sido accesi­

ble, pero no existe o simplemente se trata de una

ligadura simbólica colgada.

ENOMEM

No hay suficiente memoria disponible en el kernel.

ENOTDIR

Un componente usado como directorio en char *trayectoria de hecho, no es un directorio.




Si bien los permisos de un archivo pueden ser insertados durante la ejecución de open(), si en el transcurso de la ejecución de un programa necesitamos cambiar los permisos sobre un archivo, usaremos las funciones chmod() y fchmod(). Para ejecutar estas funciones, es muy necesario que propietario del archivo coincida con el identificador del proceso que ejecuta alguna de las dos funciones o que sea el super usuario el que ejecuta el proceso que contiene la función:



Sinopsis de chmod() y fchmod()

#include <sys/types.h>
#include <sys/stat.h>

int chmod(char *trayectoria, mode_t mode);

int fchmod(int descriptor, mode_t mode );

Devuelve 0 en caso de éxito y -1 caso de error





mode_t mode puede tomar cualquier valor de los especificados para open() como tercer parámetroe. En caso de error, errno puede tomar alguno de los valores siguientes:





Código de error

Descripción

EFAULT

char *trayectoria es una dirección inválida.

ELOOP

Encontrado un ciclo de enlaces simbólicos

ENAMETOOLONG

char *trayectoria es un nombre demasido largo

ENOENT

char *trayectoria es un archivo inexistente

ENOMEM

Imposible la asignación de memoria para los descriptores internos del nucleo.

ENOTDIR

char *trayectoria no es una trayectoria correcta

EPERM

El UID del proceso no coincide con el propietario el fichero o no es 0 (super-usuario)

EROFS

Sistema de archivos en modo lectura.




fchmod() puede devolver EBADF en caso de que descriptor no sea válido.

El UID del proceso que llama a chmod() o a fchmod() ha de ser con permisos de root o coincidir con el propietario del fichero. Debemos de tener en cuanta que en la mayoría de los sistemas el bit sticky sólo puede ser asignado por el super-usuario de manera que un fichero sólo puede ser borrado por el usuario o el superusuario.
