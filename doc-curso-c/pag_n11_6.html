11.6.1.- Creación y supresión de directorios



Al igual que cuando ejecutamos la orden mkdir() ejecutada bajo Linux crea un directorio, la función mkdir() cumple idéntico cometido:

#include <sys/types.h>

#include <fcntl.h>
#include <unistd.h>

int mkdir(char *trayectoria, mode_t mode);

Devuelve 0 en caso de éxito y -1 caso de error

11.6.2.- Cambio de directorio de un proceso, chdir() y fchdir()



Sinopsis de chdir() y fchdir()

#include <unistd.h>

int chdir(char *trayectoria);
int fchdir(int descriptor);

Devuelve 0 en caso de éxito y -1 caso de error



chdir() cambia el directorio en el que se ejcuta el proceso al especificado en char *trayectoria.

fchdir() cambia el directorio en el que se ejcuta el proceso al especificado por int fd.



Cuando se ejecuta un proceso, la identificacçon de este está asignado a un directorio determinado, normalamente al que se ejecuta. Si deseamos que se ejcute con un identificador diferente de directorio, usaremos estas dos funciones.

En caso de errror, además de devolver el valor -1 la variable errno puede adquirir los siguientes valores:



Código de error

11.6.3.- Exploración de directorios, opendir(), readir() y closedir()


Al igual que existe un tipo FILE, tambien existe un tipo DIR definido en el fichero <dirent.h>. Para poder leer un directorio, en primer lugar debemos de declarar un puntero a un directorio de la siguiente manera:


DIR *dir;



Declarando a dir como un puntero a un directorio. Para leer un directorio debemos de usar tres pasos: Abrir el directorio con opendir(), leer el directorio con la función readdir() y cerrar el directorio con closedir().


Sinopsis de opendir(), reddir() y closedir()

#include<dirent.h>
#include<sys/types.h>
#include <unistd.h>

DIR *opendir(char *trayectoria);
struct dirent *readdir(DIR *dir);
int closedir(DIR *dir);

opendir devuelve un puntero a un directorio o NULL en caso de error.

readdir devuelve un puntero a una estructura de tipo dirent o NULL en caso de error

closedir 0 en caso de éxito y -1 caso de error



Si la función opendir() no puede ejecutarse la variable errno puede tomar alguno de los siguientes valores:



Código de error

Descripción

Alcanzado el número máximo de archivos abiertos para el sistema

ENOMEM

Una vez abierto el archivo con opendir() debemos de ejecutar readdir() pasando como argumento el puntero a un directorio que hemos conseguido con opendir(). La función readdir() devuelve un puntero a una estructura de tipo dirent definida en el archivo de cabecera <dirent.h>. Esta estructura dispone de tres interesantes elementos para nosotros:



unsigned short int d_reclen;

Tamaño de la estructura devuelta

char d_name[256];

Nombre del archivo

long d_ino;

Número de inodo del archivo.
