11.1.- Detección de errores en el manejo de archivos.



Cuando intentemos abrir un archivo que no existe, ejecutar una orden imposible, o ocurra alguna deficiencia en un determinado programa que tenga que ver con el sistema, entramos en el campo de la captura de errores. Para el manejo de estos errores debemos de incluir en el archivo fuente el fichero errno.h donde está declarada la variable externa int errno. Tambien es importante declarar la variable externa extern char *_sys_errlist[]; que contiene un array de punteros a cadenas de caracteres apuntando a diferentes cadenas explicativas de los diferentes errores. No debemos confundir valores de retorno de error de las funciones con el valor que puede alcanzar la variable errno. Cada vez que una función devuelve un valor de error ( la mayoría e las veces -1) la variable errno adquiere un valor indicativo del error que se ha producido. Las diferentes cadenas de errores de que dispone Linux podemos examinarlas mediante el siguiente programa:



#include <stdio.h>

#include <errno.h>


char *_sys_errlist[];


main()

{

      printf("errno = %d\n", errno);

      for(;errno <= 124; errno++)

        printf("_sys_errlist[%d] = %s\n", errno, _sys_errlist[errno]);


}

NAME

errno - number of last error


SYNOPSIS


#include <errno.h>


extern int errno;


DESCRIPTION


The integer errno is set by system calls (and some library

functions) to indicate what went wrong. Its value is sig­

nificant only when the call returned an error (usually

-1), and a library function that does succeed is allowed

to change errno.


Sometimes, when -1 is also a legal return value one has to

zero errno before the call in order to detect possible

errors.


errno is defined by the ISO C standard to be a modifiable

lvalue of type int, and must not be explicitly declared;

errno may be a macro. errno is thread-local; setting it

in one thread does not affect its value in any other

thread.


Valid error numbers are all non-zero; errno is never set

to zero by any library function. All the error names

specified by POSIX.1 must have distinct values.


POSIX.1 (1996 edition) lists the following symbolic error

names. Of these, EDOM and ERANGE are in the ISO C stan­

dard. ISO C Amendment 1 defines the additional error num­

ber EILSEQ for coding errors in multibyte or wide charac­

ters.

Donde vemos un listado de las diferentes constantes para la variable errno. Estas constantes se corresponden con el indice correspondiente del array de cadenas char *syserrlist[].

Veamos a continuación un programa que intenta abrir un archivo que no existe y el código de errores que genera

#include <stdio.h>

#include <errno.h>


char *_sys_errlist[];


main()

  {

      FILE *stream;


      printf("errno = %d\n", errno);

      printf("Estamos en el principio del programa. _sys_errlist[errno] = %s\n", _sys_errlist[errno]);

/* intentamos abrir en modo lectura un archivo que no existe*/

      stream = fopen("ZipiZape.dat", "r");

      printf("Despues de fopen(), errno = %d\n", errno);

      printf("_sys_errlist[errno] = %s\n", _sys_errlist[errno]);

      perror("perror: ");

      printf("_sys-errlist[ENOENT] = %s\n", _sys_errlist[ENOENT]);

  }





errno = 0


Estamos en el principio del programa. _sys_errlist[errno] = Success


Despues de fopen(), errno = 2


_sys_errlist[errno] = No such file or directory


_sys-errlist[ENOENT] = No such file or directory


perror: No such file o directory



La variable externa errno es inicializada a 0. Este valor se corresponde con la cadena "success" (exitoso). Esto es comprobable con las dos primeras lineas de salida del programa. Cuando intentamos abrir un archivo que no existe en modo lectura, la variable errno adquiere el valor 2 y el indice 2 del array char *_sys_errlist[] se corresponde con la cadena: No such file or directory. Observa tambien que el 2 es sustituible por ENOENT sin ningún problema.

La función perror(char *string) muestra por pantalla el mensaje contenido entre sus paréntesis seguido del mensaje de código de error. Una vez capturado el mensaje de error debemos de situar de nuevo la variable errno a 0 ya que si no es así, no cambiará de valor pese a que se produzca un nuevo error. Esto se consigue con clearerr() o situando a mano la variable errno a 0. En el programa anterior esto lo podríamos implementar de la siguiente manera:


#include <stdio.h>

#include <errno.h>


char *_sys_errlist[];


main()

  {

      FILE *stream;


      printf("errno = %d\n", errno);

      printf("Estamos en el principio del programa. _sys_errlist[errno] = %s\n", _sys_errlist[errno]);

/* intentamos abrir en modo lectura un archivo que no existe*/

      stream = fopen("ZipiZape.dat", "r");

      perror("perror: ");

      clearerr(stream);//tambien podemos usar errno = 0

  }



Tambien podemos ver el mensaje de error con la función char *strerror(int err) usando un código similar al siguiente:

printf("El error ha sido %s\n", strerror(errno));



Resumiendo, podemos capturar mensajes de error con las siguientes funciones:

#include <errno.h>

void perror(char *mensaje);

Muestra por pantalla la cadena char *mensaje seguida del mensaje de código de error.

Compatible con UNIX, MSDOS



char *strerror(int num);

Devuelve la cadena de error resultado de trasladar int num al indice del array _sys_errlist[]




Cuando en una operación sobre un determinado archivo genera un error, puede ser capturado mediante ferror(). Para desactivar este indicador, usaremos la función clearerr().



Sinopsis ferror()

#include <stdio.h>

int ferror(FILE *stream);

Compatible con UNIX, MSDOS

Devuelve 0 si no ha generado error y un valor distinto de 0 en caso contrario



Cuando ocurre un error en una operación con un fichero determinado, el indicador de errror se pone activo (valor distinto de 0) y permanece así hasta que se ejecuta la función clearerr() pasándole a esta como parámetro el nombre del puntero al supuesto archivo generador del error.

En el manejo de archivos es muy importante la función clearerr(). Esta función recibe como parámetro el nombre del archivo susceptible de error.



Sinopsis clearerr()

#include <stdio.h>

void clearerr(FILE *stream);

Desactiva el indicador de error para el fichero FILE *stream situándolo a 0.

Compatible con UNIX, MSDOS



Una vez capturado un error con ferror() es necesario desactivar el indicador de error con la función clearerr() ya que en caso contrario ferror() no será efectiva posteriormente.

Para detectar la el fin de un fichero usamos la función feof()

Sinopsis feof()

#include <stdio.h>

int feof(FILE *stream);

Testea el indicador EOF para el fichero apuntado por FILE *stream devolviendo 0 si este carácter de finde fichero no es detectado. Devuelve un valor distinto de 0 en caso de encontrar EOF.

Compatible con UNIX, MSDOS
