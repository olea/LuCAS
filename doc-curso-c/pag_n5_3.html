5.3.- Almacenamiento auto, static y register



Todas las variables que se crean en una función son por defecto auto mientras no se especifique lo contrario. Una de las causas por la que antepongamos el identificador auto delante de nuestra declaración de variable es por que queremos ponerla así. Otra de las causas podría ser para que no se confundiera con una variable extern. Como ya hemos indicado anteriormente podríamos definir una variable auto con igual nombre identificativo que una extern. Sin embargo esta variable auto nada tiene que ver con la variable extern salvo la coincidencia de nombre identificativo. Observa:
 
 

#include <stdio.h>

int erior;

main()
    {
      auto int erior; /*sería idéntico que escribir: int erior */ 
     .
     .
    }


Las variables auto sólo se les asigna una dirección de memoria (se inicializan) cuando se ejecuta la función que las contienen. Cuando la función que las contiene finaliza, dejan su espacio de memoria libre y por tanto desaparecen. Esto consigue un ahorro de memoria que en el caso de nuestros pequeñitos programas es inapreciable y mucho más inapreciable con el potente hardware del que se dispone actualmente. En programas más extensos se conseguiran ahorros importantes de memoria. Una variable auto se puede inicializar varias veces durante la ejecución de un programa y desaparecerá de la memoria del ordenador tantas veces como se inicialicen.

Las variables auto contiene basura hata que no se les asigne un valor.
 

Variables estatic

Son variables declaradas a nivel interno. Si no se les asigna un valor C la inicializa a 0.

Aunque no lo parezca por el nombre de static estas variables pueden cambiar de valor las veces que sea necesario para el funcionamiento del programa. Recordando un poco lo anteriormente escrito, si bien las variables auto aparecen (inicializan) cuando se inicializa la función que las contiene y desaprecen cuando finaliza dicha función, las variables static se inicializan sólo una vez y no pierden ni su dirección de memoria ni su valor hasta el fin de la ejecución del programa. Por definición, las varibles static sólo se inicializan una vez. Echa un vistazo al programa:
 
 

#include <stdio.h>
int calcular(int);
main()
{
  int res, cont=1, num;
  printf("Imprime tablas de multiplicar. Escribe un número: ");
  scanf("%d", &num);
  while (cont < 11)
     {
      res = calcular(num);
      printf("%d x %d = %d\n", num, cont, res);
      cont++;
     }
  printf("\n\nastalavista");
}

int calcular(int num)
{
  static int contador=1; 
  num = num*contador++;
  return num;
}


Observa que en este didáctico programa la variable contador es declarada como static. Observa que durante la ejecución de main() se llama a la función calcular 10 veces. Observa que durante esas 10 veces la variable contador no pierde su valor y nos sirve maravillosamente a nuestros propósitos.  Sólo se ha inicializado una vez, pierde su valor cuando finaliza el programa. la salida de este programa, suponiendo que hayamos introducidoel número 2 sería:
 
 

2 x 1 = 2
2 x 2 = 4
2 x 3 = 6
2 x 4 = 8
2 x 5 = 10
2 x 6 = 12
2 x 7 = 14
2 x 8 = 16
2 x 9 = 18
2 x 10 =20

astalavista


Si no inicializamos una variable de tipo static es inicializada a 0.
 

Variables extern static

Observa que cuando declarábamos una varible extern, podíamos acceder a esta variable desde cualquier función y fichero, siempre que compiláramos juntos los ficheros que contuvieran dicha varible extern.

Cuándo declaremos una variable extern static podremos acceder a ellla sólo desde el fichero que contiene su declaración y por supuesto en funciones que sean posteriores a la declaración de la variable.
 

Variables register

Son variables declaradas a nivel local en una función. Cuándo se declara una variable como register, debería de quedar almacenada en los registros de la CPU (procesador), lugar que por razones obvias permite acceder más rápidamente a los registros de dichas variables. 
 
  

 register float num_dec;


Son iniciadas a nivel local y expiran al terminar la función que las contiene. Si has leido los párrafos anteriores, te darás cuenta que este tipo de variables se comportan exactamente igual que las variables de tipo auto, duran en memoria lo que dura la función que las contiene.
Sin embargo el determinador register es un determinador que pide dejen alojar su variable en la CPU si hubiera sitio en dicho lugar. Si no existiera ese sitio durante la ejecución del programa, quedará alojada en memoria. Ocurre que los registros de la CPU, son lugares pequeños, dónde caben poquitos registros de variables. La CPU los concede a quien se lo pida si tiene el sitio disponible, caso contrario lo envía de vuelta a la RAM. Por si fuera poco estos interesantes y pequeños lugares son muy solicitados por los multitud de procesos que corren en un computador. No resulta descabellada la forma de actuar de register. Para podernos hacer una idea del alcance de ciertas variables, observa la siguiente tabla:
 
 

Tipo de almacenamiento

Alcance

Duración

extern 

Todas las funciones y todos los ficheros que componen el programa

La función que la contiene

Lo que dura la función que la contiene

El atento lector se preguntará porqué le hemos complicado la vida de esta manera. Pensais que a partir de este instante, cuando se os ocurra algún programa, el momento de la declaración de variables será un momento particularmente desagradable ya que deberemos de saber que tipo de variable emplear. Sin embargo esto no es así. Como habrás visto en los programas que hemos desarrollado hasta el momento, casi todas las variables son de tipo auto. A medida que empieces a desarrollar porgramas más y más grandes, comprenderás el uso de este tipo de definidores. Es más, símplemente se te crearán necesidades imperiosas de declarar ciertos tipos de variables como extern o static.

 A estas alturas no te preocupes mucho, usa tus variables como auto para comenzar. Sólo usa otro tipo de variable distinta a auto cuando creas que la situación lo requiere. Es importane que asumas este capítulo, además de saber utilizar las variables en C, te servirá para más adelante entender el porqué de ciertas denominaciones en Java y C++.
 
 

¡¡¡¡Importante!!!!!

  Si quereis hacer vuestro programa portable de verdad de un sistema a otro, es importante que os olvideis por completo de los acentos. Programas sin errores en un sistema, originan un montón de errores durante la compilación en un sistema diferente. Tantos errores como acentos incluso si estos acentos están en los comentarios.

Es mucho más interesante hacer programas "universales", que funcionen en diversos sistemas operativos. Para eso se creó C y otros lenguajes. Lamentablemente ciertos sistemas operativos disponen de una filosofía completamente opuesta, intentando hacer y desarrolllar herramientas lo más incompatibles posibles.
