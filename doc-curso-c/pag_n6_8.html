6.8.- while y do                                           





La sintaxis de while() es:


/* En el caso de que haya sólo una sentencia se pueden omitir las llaves */
while(condicion)
         { 
          sentencias
         }





Ya hemos visto varios ejemplos dónde usamos el bucle while(). Hemos de tener cuidado con este bucle, pues si hacemos un deficiente diseño de su estructura podríamos encontrarnos con que se ejecuta tan deprisa que no lo veremos en la pantalla (bucle infinito).

La palabra while() significa en español 'mientras'. Podríamos traducirlo como que 'mientras' sea verdadera la condición expresada en el interior de los parentesis, se ejecutan las sentencias contenidas que yacen en el interior de las llaves si es que las tiene.


La sentencia do, está directamente emparentada con while(), pues desarrolla las sentencias contenidas entre sus llaves 'mientras' la condición de la sentencia while() sea verdadera. Esta es la sintaxis de do:
 

do
     {
       /* sentencias */
     }while(condicion);


Observemos un fácil ejemplo de do:
 

#include <stdio.h>

main()
  {
  int num;

  printf("Este programa halla cuadrados superioresa 99. Para salir del programa introduce un número menor que 100\n\n\n");

  do
    {
    printf("Escribe un número superior a 100 para hallar su cuadrado: ");
    scanf("%d", &num);
    printf("El cuadrado de %d es %d\n\n", num, num * num);
    } while(num > 99);

printf("ha sido un placer\n");
}
 


Si ejecutamos este programa solicitando el cuadrado de cualquier número menor que 100, observa lo que ocurre:
 
 

Escribe un número superior a 100 para hallar su cuadrado: 5
El cuadrado de 5 es 25

Ha sido un placer



Date cuenta de lo extraño del suceso. Hemos asignado a num el número 5, que es bastante más pequeño que 100. Sin embargo la condición para que  se ejecute  while() es que el valor de su interior sea mayor que 99 y nosostros hemos metido un valor inferior, a pesar de eso nos da el valor del cuadrado de 5 y luego sale del bucle y finaliza el programa. La razón del suceso es que la sentencia do se ejecuta en primer lugar que while(). Despues while() decide si volver a ejecutar do o continuar de nuevo con do. Puedes probar a compilar el programa eliminando la sentencia while(), verás que do se ejecuta perfectamente.
