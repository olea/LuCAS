7.5.- Haciéndonos sitio a codazos en la memoria. malloc() y calloc()


Anteriormente hemos usado arrays de punteros a cadenas de caracteres y hemos visto la comodidad que ello supone, incluso el ahorro de memoria que conseguimos. Nunca hemos intentado rellenar un array de punteros a cadenas de caracteres con la función gets() o scanf(). Para ello intentaremos hacerlo con los datos que conocemos hasta ahora. Vamos a intentar asignar la dirección de una cadena de caracteres a un puntero a char. De hecho no deberiamos de tener problemas, pues hemos visto anteriormente que el nombre de una array de caracteres es la dirección del primer elemento del array. Observemos atentamente el sencillo ejemplo siguiente:





/*El código de este programa genera error en la ejecución*/

/******** cadeneta.c *********/

#include <stdio.h>

void main()
{
  char *c;

  printf("Escribe a continuación una cadena de caracteres: ");
  gets(c);
  printf("Has escrito la cadena \"%s\"\n", c);
}






Este programa debería de funcionar, declaramos el puntero *c a char, a continuación capturamos una tira de caracteres con gets() y lo mostramos por pantalla. Sin embargo no funciona. Este es el resultado del programa:

Escribe algo: Programo en C
segmentation fault


Segmentation fault significa que falta segmentación, falta continuación del programa y esto es porque en nuestro progama se ha sobreescrito sobre su propio código, con lo que el programa no encuentra el camino para terminar su ejecución. Ocurre que la función gets() hace lo que le ordena el programa y el programa le está ordenando escribir ciertas cadenas de texto introducidas por el teclado. Ni gets() ni scanf() comprueban que hay escrito en el lugar que van ellas a sobreescribir. Por eso gets() elimina código previamente escrito por el programador. De hecho el programa dispone de un ámbito de bytes determinado, si intentamos escribir en dicho ámbito, y no tenemos sitio reservado para este fin nuestro programa escribirá sin importarle dónde escribe.

En el programa cadeneta.c observamos que tenemos declarado un puntero char *c este puntero tiene capacidad de apuntar a cualquier cosa que sea un carácter de teclado, puede apuntar a un carácter o al primer carácter de una cadena de caracteres. Observa que en el programa sólo hemos reservado espacio en memoria para un puntero, pero para nada más. Para poder crear un espacio de momeria en el que podamos almacenar una cadena de caracteres, un array, etc debemos de usar la función malloc(). Esta función devuelve una dirección de memoria (puntero) a la varaible puntero a char c. A continuación puedes observar la manera de usarla:



NOMBRE

malloc() - Asignador de memoria


SINOPSIS

include<stdlib.h>


void *malloc(size_t size);



Si la asignación de memoria no se ha podido cumplir, devuelve un puntero nulo NULL. Supongamos que queremos que queremos asignar espacio en memoria para una cadena de 20 caracteres. La asignación la haríamos de la siguiente forma:

c = (char *)malloc(21);




El término (char *) es una operación de casting para indicar al valor devuelto que es un puntero a char. Esta declaración reserva un espacio de memoria de 21 bytes en un determinada dirección e memoria, esa dirección es asignada al puntero a char c. Observa que hemos reservado un byte más del que necesitamos, es precisamente para alamacenar el carácter nulo de fin de cadena '\0'. En este punto ya podremos usar la función gets(c), que alamacenará la cadena introducida por teclado en la dirección que contiene c. Una vez que no necesitemos el contenido de la dirección apuntada por c, hemos de ejecutar la función free() con el fin de que el espacio asignado pueda ser usado de nuevo. Piensa que en aplicaciones grandes la no liberalización de la memoria puede originar un colapso del sistema debido a la ocupación integra de la memoria por el programa. Una vez finalizado el programa la memoria reservada queda libre para el uso que se quiera dar. En todo caso es muy buena costumbre de liberar la memoria con la función free(). Veamos su definición:

NOMBRE

free() - Libera memoria asignada dinámicamente


SINOPSIS


include<stdlib.h>

void free (void *ptr);


Para liberar la memoria asignada al puntero c por el programa lo haríamos de la siguiente manera:

free (c);


Ya sabemos como asignar la memoria y liberarla. A la operación de la asiganación de memoria la denominamos a asignación dinámica de memoria. El programa quedaría de la siguiente manera:


/******** cadeneta.c *********/


#include <stdio.h>


void main()

  {

   char *c;


   printf("Escribe a continuación una cadena de no más de 20 caracteres: ");


   /* Asignamos 20 bytes de memoria */

   c = (char *)malloc(21);

   /* Colocamos el carácter fin de cadena en la posición 20 */

   gets(c);

   printf("Has escrito la cadena \"%s\"\n", c);

   /* Liberamos la memoria asignada con malloc() */

   free(c);

  }




Seguro que ahora el programilla funciona, veamos la salida del mismo:

Escribe a continuación una cadena de no más de 20 caracteres: Programo en C
Has escrito la cadena "Programo en C"




Asignación dinámica de memoria en arrays de cadenas



Anteriormente vimos como asiganar cadenas en arrays "rectangulares" (arrays cuyas filas son de idéntica longitud), con la cosiguiente pérdida de memoria que esto constituía. Tambien hemos inicializado cadenas constantes de caracteres usando la forma

char <nombre de cadena>[] = "<cadena>"

Incluso hemos usado punteros que inicializamos apuntando a una cadena de la forma:

char *<nombre puntero> = "<cadena>"

Evidentemente para la asignación de una cadena de caracteres el desperdicio es mínimo, el problema surje con los arrays de cadenas de caracteres. A fin de que asignemos la memoria necesaria para ello usaremos la función malloc(). Desarrollaremos un programa que nos permita asignar dinámicamente memoria para almacenar cedenas de caracteres en las direcciones apuntadas por un array de punteros a cadenas. Para ello estimamos que cada cadena no excederá de 80 caracteres y como máximo nos va a permitir almacenar 100 cadenas. El programa podría ser algo parecido a esto:

Para almacenar datos en las direcciones del array de punteros lo hacemos indirectamente. Para ello primero declaramos una array de caracteres de longitud máxima 81 (linea 8) , realmente es 80, pero usamos el carácter nº 81 para guardar el carácter de fin de cadena \0. Una vez almacenada la cadena introducida por teclado en cad[81], mediremos su tamaño. Cad[81] siempre tendrá 81 bytes destinados para esto, pero una cadena almacenada en su interior puede medir menos, de hecho medirá desde cad[0] hasta el carácter '\0'. Para ejecutar estas mediciones usamos if(strlen(gets(cad)) == 0. Esta línea hace varias trabajos: En primer lugar captura una cadena por el teclado, despues averigua su longitud y si esa longitud es 0 (caso que el usuario pulse <enter> sin haber introducido algún carácter), entonces entoces da las gracias y el flujo del programa sale del bucle for(). La asignación de espacio de memoria la realizamos mediante



24

       cads[i] = (char *)malloc(strlen(cad) + 1);



Observa que al igual que en el ejemplo anterior reservamos un espacio más de memoria para alojar el carácter fin de cadena en el caso de que la cadena almacenada tenga 80 caracteres.
A continuación usamos la función strcpy(cads[i], cad). Esta función copia los caracteres apuntados por cad (incluyendo el carácter nulo) en la dirección apuntada por cads[i]. Observa que tambien podríamos haber asignado el valor de cad a cads[i] usando la forma: cads[i] = cad, pero esto no funcionaría, símplemente copiaría la dirección de memoria de cad en cad[i], con lo que tendráimos varios punteros apuntando a la misma dirección de memoria, y esta dirección de memoria contendrá datos diferentes en la siguiente iteración de for(). Si introdujéramos 50 cadenas de caracteres, conseguiriamos un array de 50 punteros a cadenas de caracteres apuntando al mismo sitio.


Incluso en la cadena cad[81] podemos declararla como un puntero a char y despues asignar memoria para ella. Además mejoraremos nuestro programa dando un mensaje de error al usuario cuando no exista memoria disponible tanto en la reserva de memoria para *cad como para *cads. Función calloc()


NOMBRE

calloc() - Asigna memoria dinámica para arrays


SINOPSIS

include<stdlib.h>

void *calloc(size_t nmemb, size_t size);



Dónde size_t nmemb es el número de células de memoria requerido y size_t size es el tamaño de cada célula medida en bytes. Para crear memoria para un array de 10 elelemtos int podríamos usar calloc() de la siguiente manera:

#include<stdio.h>
#include<stdlib.h>

void main()
{
  int *p, i;
  p = (int *)calloc(10, sizeof(int));

  for(i=0; i<5; i++)
   {
    printf("Escribe un número: ");
/* Observa que por primera vez no usamos el operador dirección '&' en scanf() */
    scanf("%d", p+i);
   }
  for(i=0; i<10; i++)
  printf("*(p + %d) = %d\n", i, *(p + i));
}


La declaración de calloc (int *) es el tipo de valor al que apunta la dirección devuelta , 10 es el número de elementos del que consta el array y sizeof(int) es el tamaño de un elemento de tipo int en el sistema qua estamos usando. Podriamos usar el número 4, que es lo que ocupa un int en un sistema Linux o un 2 que es lo que ocupa un int bajo MSDOS, pero al usar sizeof(int) nos evitamos tener que investigar en qué plataforma se va a lanzar nuestro programa. Fíjate que en la función scanf("%d", p+i) no usamos el operador dirección de memoria '&'. Y no lo usamos porque p ya es una dirección. La función scanf() necesita como segundo argumento una dirección de memoria y a sea la dirección de una variable o la dirección de memoria almacenada en un puntero. Si usáramos la fórmula scanf("%d", &p), en la dirección de p almacenaríamos un elemento de tipo int, y la dirección de p sólo puede contener direcciones de memoria que apunten a elementos int, con lo que nuestro porgrama generaría un error de compilación. La salida del programa será:

Escribe un número: 10
Escribe un número: 20
Escribe un número: 30
Escribe un número: 40
Escribe un número: 50

*(p + 0) = 10
*(p + 1) = 20
*(p + 2) = 30
*(p + 3) = 40
*(p + 4) = 50


Otra particularidad de la función calloc() es que inicializa todos sus elementos a 0.
