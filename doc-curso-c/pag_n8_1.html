8.1.- Estructuras                                   





Supongamos que queremos acoplar en un programa diversos animales para clasificarlos. Nuestros animales quedarán definidos por el número de patas, el tipo de pelaje, si es mamífero o no lo es, si tiene pico o no y la familia a la que pertenece. Es evidente que los animales tienen muchas más características, de hecho podríamos escribir un libro sobre este tema, pero a nosotros sólo nos interesa destacar estos puntos. Veamos como podemos definir una variable de tipo animal:
 


struct animal

{
        char nombre[20];
        int n_patas;
        char pelaje[10];
        int pico;
        int mamifero;
        char familia[20];
};


En C decimos que hemos declarado la estructura animal. De hecho hemos creado un tipo de variable. A partir de aquí podemos declarar variables de tipo animal. De momento en esta declaración de la estructura animal no podemos introducir valores para sus variables internas. Podríamos intentar hacer lo siguiente:
 


/* Esta inicialización de n_patas dará error de compilación */ 

struct animal

{
      char nombre[20];

      int n_patas = 4; //Esto es ilegal

      char pelaje[10];

      int pico;

      int mamifero;

      char familia[20];

};


Sería algo así como intentar asignar un valor a int sin declarar un tipo de variable, esto es un modelo de variable y hasta que no declaremos una variable de tipo animal no podremos introducir datos.

Supongamos que en nuestro programa entra un aguila y queremos aplicarle las características de la variable animal. Esto lo haremos de la siguiente manera:
 


struct animal aguila



Ahora ya tenemos declarada una variable aguila. Para llegar a este punto, C tiene almacenado en algúna parte de su memoria la estructura animal que ocupa 62 bytes (si usas Windows9x ocupará 56 bytes), este espacio de memoria lo usará como modelo para futuras declaraciones de variables de tipo animal. Cuando hacemos la declaración struct animal aguila, al ejecutar el programa tomará como modelo la estructura animal y estampará idéntico espacio de memoria en algún lugar que esté libre para almacenar los 62 bytes que necesita para ello. Esta variable se llama aguila y el contenido de sus variables internas contine basura, por ello hemos de rellenar dichas variables internas a aguila. Para ello usaremos el operador de pertenencia.

El operador de pertenencia.- Para referirnos a las variables internas de una estructura usamos este operador que se representa por un punto: '.'. Lo usaremos usando la notación

<nombre de variable estructura>.<variable interna de la estructura>

Para referirnos a los elementos de aguila lo haremos de la siguiente manera:
 


aguila.nombre // Esto es un puntero a una cadena de caracteres
aguila.n_patas; // Esto es un int
aguila.pelaje; // Esto es un puntero a una cadena de caracteres aguila.pico; // Esto es un int
aguila.mamifero; // Esto es un int
aguila.familia; // Esto es puntero a una cadena de caracteres


Estos elementos se tratan exactamente igual que el tipo de variable al que petenecen. Ahora vemos en el siguiente programa como declaramos la variable aguila de la estructura animal. Rellenaremos sus variables internas y las veremos por pantalla:
 


#include <stdio.h> 


#define SI 1

#define NO 0

void main()

      {
              /*Declaración de una estructura tipo*/

         struct animal

               {
                char nombre [20];

                int n_patas;

                char pelaje[10];

                int pico;

                int mamifero;

                char familia[20];

              };


             /* Declaramos la variable aguila de tipo animal */
        struct animal aguila;
/* Asignación de datos a las variables internas de aguila */

        strcpy(aguila.nombre, "aguila");
        aguila.n_patas = 2;

        strcpy(aguila.pelaje, "plumas");

        aguila.pico = SI;

        aguila.mamifero = NO;

        strcpy(aguila.familia, "rapaces"); 

              /* Visualizamos los datos de aguila */

        puts("DATOS DE AGUILA\n");

        printf("Nombre: %s\n", aguila.nombre);

        printf("nº patas: %d\n", aguila.n_patas);

        printf("Tipo de pelaje: %s\n", aguila.pelaje);

        if (aguila.pico == SI)

                 printf("Pico: SI\n");

        else

                 printf("Pico: NO\n");

        if (aguila.mamifero == SI)

                 printf("Mamifero: SI\n");

        else

                 printf("Mamífero: NO\n");

        printf("Familia: %s\n", aguila.familia);

}


 
 

Si intentas asignar datos a animal te encontrarás que el compilador no te deja hacerlo, el programa no compila, es imposible. La razón es que animal es un modelo de estructura, de alguna manera es el nombre de un tipo de variable. Si intentáramos hacer algo parecido a esto:
 


animal.patas = 2 //esto genera un error de compilación


Veamos la salida del programa:


DATOS DE AGUILA

Nombre: aguila


nº patas: 2

Tipo de pelaje: plumas

Pico: SI

Mamífero: NO

Familia: rapaces
