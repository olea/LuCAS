7.2.- Cadenas de caracteres y punteros


Desde este momento pido enormes disculpas al observador y considerado lector sobre la repetición de diversos problemas respecto a direcciones de memoria y punteros, probablemente a alguien le resulte tedioso y repetitivo debido a que lo ha captado al primer vistazo, a otras personas necesitarán más explicaciones. El asunto es que los punteros y direcciones de memoria es lo que hace de C un lenguaje especialmente interesante y a partir de este capítulo los punteros aparecerán con una frecuencia apabullante. De hecho en cualquier código fuente que llegue a tus manos dispondrá de estos elementos. Es fundamental para avanzar en C comprender este capítulo lo mejor posible, esto nos dará una sólida base para avanzar en lo que queda de temario. Por si fuera poco, esto servirá de gran ayuda para afrontar otros lenguajes de programación.


La mayoría de los estudiosos de C cuando nos hemos encontrado con los punteros por primera vez pensamos: "uuuuuuuffffffff!!!! esto ya lo miraré, me puedo referir a un array sin usar punteros, luego me las voy a arreglar bien sin los punteros, ya los miraré cuando tenga tiempo". Con esta actitud lo único que hacemos es posponer lo inevitable, en algún momento vamos a tener que afrontarlos. En todo caso esto que te estoy explicando es voluntario, si quieres lo miras y si no pues no. Pero volverás a este capítulo. El panadero de mi barrio me dijo una vez que los punteros son a C, como la levadura al pan. Empecemos.




7.2.1.- Las funciones gets() y puts()




Sinopsis gets()

#include <stdio.h>
char *gets(char *cadena);

Compatibilidad MSDOS y UNIX
Devuelve un puntero a la cadena de caracteres leida




Sinopsis puts()

#include <stdio.h>
int *puts(char *cadena);

Compatibilidad MSDOS y UNIX
Devuelve un valor positivo si de ejecuta satisfactoriamente. NULL en caso contrario




Para leer y mostrar cadenas de caracteres vamos a usar las funciones gets() y puts(). La función gets() permite introducir una cadena de caracteres en un array de tipo char. puts() mostrará por pantalla la cadena elegida. Dichas cadenas de caracteres son tratadas por C como arrays de variables de tipo char. Como ya vimos en el anterior capítulo, en el nombre del array está contenida la dirección de memoria del primer elemento del array. Para que el programa sepa dónde termina el array, en todas las cadenas de caracteres se añade al caracter nulo ('\0') al término del array para indicar al copmpilador donde termina. En todo caso esto es trasparente para el programador a no ser que dicho programador disponga lo contrario. Veamos una ejemplo de como trabaja gets(), puts() y como añade al final de las cadenas el caracter nulo ('\0'):

Empecemos con el array saludo. Como asunto interesante podemos observar la linea 4:


4

     char saludo[] = "hola";


Notamos que el número de elementos del array saludo está omitido y esto es porque el compilador cuenta el número de caracteres del array y una vez contados reserva espacio consecutivo de memoria para alojar los elementos del array, la cadena hola. Cuando el programa comience, tendremos un espacio de memoria reservado en el que está alojadola cadena designada por el array saludo.


En la linea 5 hemos declarado la longitud de nombre como de 81 debido a que es el número de caracteres que ocupa una línea en modo consola más una unidad. Más tarde veremos para que nos sirve ese caracter de más que situamos. En todo caso a estas alturas de programa, ya tenemos reservado en la memoria un espacio de 81 elementos de tipo char que está esperando a poderse llenar. En principio el programa espera que nombre aloje el nombre del usuario del programa. Para rellenar este array podremos usar la función scanf() de la forma:
 

scanf ("%s", nombre); 


Te prometemos que no nos hemos olvidado de poner el símbolo '&', lo que ocurre es que no se pone. Observa que el operador '&' es un operador que devuelve una dirección. Lo situábamos delante de nombres de variables numéricas para que conociera el programa dónde alojar el valor introducido por el teclado, sin embargo el nombre de un array es la dirección de memoria del comienzo del mismo, un puntero a char, luego no nos hace falta situar la dirección, pues el mismo nombre del array ya es una dirección de memoria. A pesar de todo la función scanf() no nos hubiera rellenado el array nombre[81] tan perfectamente. Si utilizáramos para este menester la función scanf(), almacenaría en el array el nombre introducido por el teclado hasta el primer espacio en blanco tecleado. Esta es una cualidad de scanf(). Nos podría valer para introducir el nombre Locomotoro, pero si deseáramos introducir la cadena Locomotoro Kabezabolo no valdría, pues existe un espaciado en blanco en la cadena y sólo leería la cadena Locomotoro ignorando el espacio en blanco y Kabezabolo. Par leer una cadena de caracteres que contenga espacios en blanco usamos para ello la función gets(), ya que desconfiamos de que el usuario de nuestro programa se limite a sólo escribir su nombre. Ciertos usuarios van a pretender introducir su nombre y apellidos y para ello la función gets() nos conviene bastante, es más cómoda y sirve tanto para cadenas con espacios en blanco como sin ellos, además y aunque no te lo parezca es más simple de usar.

Para usar la función gets() no hemos más que invocarla e introduciendo entre sus paréntesis el nombre del array que queremos rellenar tal como hemos hecho en nuestro programa. Y el array nombre se comenzará a llenar hasta que pulsemos la tecla [enter]. Como hemos especificado un tamaño de 81 elementos, el usuario del programa no deberá de introducir más de 80 caracteres. Si esto ocurriera el programa responderá algo así como segmentation fault. Y esto es debido a que nos hemos excedido en la capacidad del array.

Para mostar cualquier cadena de carcteres podemos usar la función printf() símplemente especificando el tipo de dato que vamos a mostrar por pantalla y ya vimos en las especificaciones de printf() en el capítulo 4 como para mostrar una cadena de caracteres se usaba la notacion %s de la forma:


printf("%s", <nombre de array>);


Usando la función puts() tan sólo hemos de especificar la dirección de memoria que deseamos mostrar por pantalla y la dirección de memoria está contenida en el nombre del array que queremos visualizar. Sencillo? Ciertamente lo es. La forma de hacerlo es:


puts(<dirección de memoria nombre de array>)


Contempla como para visualzar la cadena "hola" utilizamos la instrucción puts(saludo). Tambien podrías haber dispuesto lo siguiente con idénticos resultados:



puts(&saludo[0]);


es equivalente a



puts(saludo);


La primera notación es válida, pero es un poco feucha de ver y un tanto rebuscada, normalmente usaremos la segunda. Tambien hemos mostrado una cadena de caracteres en crudo,  sin hacerlo a través del nombre de un array. Para ello tan sólo hemos tecleado entre comillas la cadena que queremos visualizar de la forma:




puts("<cadena a visualizar>");


La función printf() hubiera resuelto la situación de forma idéntica:



printf("<cadena a visualizar>");


¿Que diferencia de resultados tienen laa función printf() y la función puts() ? La diferencia radica en que puts() añade el caracter nueva línea al final de cada cadena a visualizar. A partir del momento en que visualicemos por pantalla una cadena de caracteres mostrada por la función puts() la siguiente visualización por pantalla se hará en la línea siguiente. Para que podamos desarrollar este efecto con la función printf() no nos queda más remedio que añadir al final de la cadena que pretendemos mostrar por esta función el caracter '\n'.

Como ya vimos anteriormente, efectivamente el tamaño de cada elemento del array es de 81 bytes. Son 4 cadenas de 81 elementos cada una. deportes es el nombre del array y al igual que los demás arrays contiene una dirección de memoria por tanto decimos que deportes es un puntero a un array de cadenas de caracteres. Para visualizar una cadena del array deportes tan sólo hemos de elegir una de las funciones siguientes:

printf("%s", deportes[<numero de cadena a visualizar>]);

o

puts(deportes[<número de cadena a visualizar>]);

Tambien podemos ver que deportes, &deportes[0], &deportes[0][0] tinenen el mismo valor. Esto es porque el puntero a char deportes, apunta al primer elemento del array que es deportes[0] y la dirección de deportes[0][0] es la misma que deportes[0]. Y en este array de cadenas ocurre algo similar a los arrays de caracteres. Si sumamos una unidad al nombre del array deportes el resultado es el valor de la dirección más el numero de columnas del array, de hecho estamos sumando 1 elemento más del array. Observa las siguientes igualdades:

deportes + 1 == &deportes[1]
deportes + 1 == &deportes[1][0]

Además observa que el puntero deportes vale 3136 y deportes + 1 es 3217. Cuando sumamos 1 unidad a deportes, estamos sumando el valor de una fila. Cada fila mide 81 bytes, es lógico pensar que si estamos sumando direcciones de memoria estamos sumando en este caso filas de memoria que es lo mismo una fila. Por tanto si ejecutamos la operación

(deportes + 1) - deportes

obtendremos como resultado el tamaño de cada fila que es 81.

Cuando usamos #define estamos usando una constante, es una instrucción para el preprocesador de C. No podremos variar a lo largo del programa el contenido de FIRMA. Veamos como funciona el array saludo[]. Si contamos los caracteres encerrados entre comillas, podemos comprobar que contiene 4 caracteres. Sin embargo, si ejecutamos la sentencia sizeof(saludo) obtendremos el resultado de 5, un caracter más de los que hemos contado nosotros. Nosotros hemos contado bien y el compilador tambien. Ocurre que cuando el compilador se encuentra una cadena de caracteres encerrada entre comillas realiza varios pasos. En primer lugar cuenta el númerode caracteres contenidos entre las comillas, al resultado de la cuenta suma 1. Con esta suma reserva un espacio extra para alojar el caracter nulo ('\0'), caracter que sitúa al final de la cadena de caracterres. Este curioso caracter sirve para que durante el desarrollo del programa se conozca la dirección donde termina el array y precisamente terminará cuando lea el caracter '\0'. La dirección donde comienza el array viene especificada por el nombre del array que como ya indicamos en el capítulo anterior que contiene la dirección de memoria del primer elemento del array. Fijate qué fácil, el programa ya sabe dónde empieza y termina saludo[]. Va a ser muy dificil que se pueda equivocar. Observa que el array saludo tambien lo podríamos haber inicializado de la siguiente manera:
 

char saludo[]={'h', 'o', 'l', 'a', '\0'};


Observa el caracter nulo que situamos como último elemento del array, si esto no fuera así cuando intentáramos leer el array saludo el programa comenzaría a leer memoria hasta que se encontrara el caracter nulo escrito en algún recóndito rincón de la RAM.

Para ilustrar más el contenido de lo expuesto vamos a echar un vistazo al siguiente programa tan inútil como instructivo:

Este array de cadenas contiene 4 punteros a cadenas de carateres. El primer elemento del array seeá por definiciónes anteriores poema[0], el segundo será poema[1], etc. Si representáramos las direcciones de memoria mediante la intersección de filas con columnas los valores de nuestro array serían:

La dirección de poema[0] es (1, a) y contiene una 'A'
La dirección de poema[1] es (28, a) y contiene una 'e'
La dirección de poema[2] es (11, c) y contiene una 'S'
La dirección de poema[3] es (3, e) y contiene otra 'S'


Si llamamos por ejemplo a la función puts(poema[2] ); el programa se irá a buscar la dirección de memoria alojada en poema[2] que es la (11, c) y que se corresponde con la letra 'S'. A partir de aquí comenzará a leer caracteres hasta encontrar un carácter nulo ('\0'), por tanto leerá todos los caracteres contenidos en el rango (11, c - 1,e) . De nuevo destacar que los caracteres nulos '\0' son añadidos automáticamente por el compilador.

Date cuenta que el array de punteros a cadenas de caacteres no contienen las cadenas en sí mismas, sino las dirección de comienzo de cada cadena. Si esto fuera así al ejecutar la sentencia int tamanyo = sizeof(poema), deberíamos de obtener para tamanyo un valor cercano a 150, que es aproximadamente el valor del número total de caracteres del poema. Esto no es del todo cierto. Cuando el programa arranca busca espacio en la RAM para alojar el array y una vez almacenado asigna a los elementos del array los valores de las direcciones de comienzo de cada una de las cadenas de caracteres almacenados en algún lugar de la memoria. La realidad es que tamanyo va a adquirir el valor de 16, valor que nos significa que el array poema dispone de 4 elementos pertenecientes a 4 direcciones de memoria. El array poema es un array de direcciones de memoria y cada una de ellas apunta a la comienzo de las distintas cadenas de caracteres que queramos almacenar. Realmente el array podríamos imaginarlo commo algo parecido a esto:

Si te entretienes en contar el número de caracteres del primer verso observarás que equivale a la resta del primer elemento menos el segundo . Esto significa que tenemos absolutamente usada la memoria que nos ocupan los 4 versos del poema. Al igual que en el programa mis_deportes.c, podríamos haber declarado el array de la manera siguiente, de 5 versos y 81 columnas con el consiguiente desaprovechamiento de memoria:
 

char poema[VERSOS][81] = {
"A UNIX sales lady, Leonor,",
"enjoys work, but she likes the beach more.",
"She found the good way to combine the work and play",
"she sells C shells by the seashore";

Donde 81 sería la longitud de una linea en consola. Este array ocuparía en memoria VERSOS * 81 que es lo mismo que 4 * 81 y que equivale a 324 bytes, donde tendríamos desperdiciados unos 200 bytes de memoria. Ocurre que cada verso sería almacenado en un array de 81 elementos de tipo char y todo el espacio restante de la longitud de nuestro verso (cada caracter del verso es un byte) hasta 81 será desperdiciado. Es más práctico usar la forma de cadenas de punteros a char.

Podríamos introducir caracteres pulsados por el teclado en un array con la función getchar() y mostrarlos por pantalla mediante la función putchar(). Veamos un ejemplo:

#include <stdio.h>
main()
  {

i++;

       putchar(cadena[i]);

   putchar(cadena[i]);
}
