11.3.- Creación de enlaces, enlaces duros y simbólicos



Si bien a topdos los nombres de archivos les coresponde un número de inodo, cuando existe más de un nombre para referirse a un archivo - inodo-, decimos que el archivo en cuestión dispone de enlaces duros. Imaginemos que disponemos de un archivo llamado Apache cuyo número de inodo es 2700. Si quremos crear un enlace duro hacia el archivo Apache con el nombre Comanche ejecutaremos el siguiente comando en la consola de Linux:


ln Apache Comanche



Si a continuación ejecutamos el comando ls -i sobre el directorio en los que se encuentran esos enlaces, veremos que tanto Apache como Comanche disponen del mismo número de ínodo. El borrado de Apache no implicará el borrado de Comanche y viceversa.

Si ejecutamos en la consola de Linux el comando


ln -s Apache Comanche



Habremos creado un enlace llamado simbólico sobre el archivo Apache. Realmente hemos creado un puntero al archivo Apache, el nombre del puntero es Comanche. Cuando borremos Apache, borraremos todos sus enlaces.

El sistema Linux dispone de funciones para crear enlaace duros y simbólicos. Empecemos con los duros:





11.3.1.- Creación de enlaces duros





Sinopsis de link()

#include <unistd.h>
int link(char *trayectoria_original, char *nueva_trayectoria);

Devuelve 0 en caso de éxito y -1 caso de error



En caso de error la variable errno puede tornar los siguientes valores:



Código de error

Descripción

EACCES

Acceso especificado no permitido

EEXIST

char *nueva_trayectoria ya existe

char *trayectoria_original ó char *nueva_trayectoria son trayectorias inválidas

ELOOP

Encontrado un ciclo de enlaces simbólicos

Alcanzado el número de enlaces máximo

ENAMETOOLONG

char *trayectoria_original ó char *nueva_trayectoria son nombres demasidos largos

ENOENT

char *trayectoria_original no existe

ENOMEM

Imposible la asignación de memoria del nucleo para descriptores internos

ENOSPC

Sistema de archivos saturados.



ENOTDIR

char *trayectoria_original ó char *nueva_trayectoria no tienen correctamente especificados las diferentes trayectorias.

EPERM

char *trayectoria_original ó char *nueva_trayectoria no soportan la creación de enlaces o talvez char *trayectoria_original hace referencia a un directorio

EROFS

Sistema de archivos en modo lectura y se ha especificado el modo escritura

EXDEV

char *trayectoria_original ó char *nueva_trayectoria se encuentran en sistemas de archivos diferentes







11.3.2.- Creación de enlaces simbólicos, symlink() y readlink()





Sinopsis de symlink()

#include <unistd.h>

int symlink(char *trayectoria_original, char *nueva_trayectoria);

Crea un enlace simbólico hacia el archivo designado en char *trayectoria_original

Devuelve 0 en caso de éxito y -1 caso de error



Este tipo de nelaces son tambien llamados enlaces blandos. Incluso puede crearse un enlace simbólico sobre un archivo inexistente. En este caso se denominan enlaces colgados - dangling links-. Los permisos que adquieren estos enlaces son irrelevantes - de hecho son creados con todos los permisos - pues el control de un archivo en lo tocante a permisos comienza en la apertura misma del archivo.

La variable errno puede tomar los siguientes valores:



Código de error

Descripción

EEXIST

char *nueva_trayectoria ya existe

EFAULT

char *trayectoria_original ó char *nueva_trayectoria son trayectorias inválidas

ELOOP

Encontrado un ciclo de enlaces simbólicos

ENAMETOOLONG

char *trayectoria_original ó char *nueva_trayectoria son nombres demasidos largos

ENOTDIR

char *nueva_trayectoria es incorrecto

ENOMEM

Imposible la asignación de memoria del nucleo para descriptores internos

ENOSPC

Sistema de archivos saturados.

EPERM

El sistema donde yace char *nueva_trayectoria no soportan la creación de enlaces o tal vez se carece de permiso de escritura en el directorio especificado por char *nueva_trayectoria

EROFS

Sistema de archivos sólo es en modo lectura


Para averiguar el nombre del archivo al que apunta un enlace simbólico usamos la función readlink()



Sinopsis de readlink()

#include <unistd.h>

int readlink(char *trayectoria, char *buffer, size_t bufsize);

Devuelve 0 en caso de éxito y -1 caso de error



Esta función permite leer el el archivo al que apunta número char *trayectoria en char *buffer. size_t bufsize especifica el número de bytes que se quiere almacenar en el array apuntado por char *buffer. readlink() no inserta el carácter nulo de fin de cadena en char *buffer, por lo quue tenemos que hacer "a mano" dicha insercción.

Los valores para la variable errno pueden ser:



Código de error

Descripción

EFAULT

char *trayectoria ó char *buffer son direcciones inválidas

EINVAL

char *trayectoria no es un enlace simbólico

EIO

Error de entrada/ salida.

ELOOP

Encontrado un ciclo de enlaces simbólicos

ENAMETOOLONG

char *trayectoria es un nombre demasido largo

ENOENT

char *trayectoria es un archivo inexistente

ENOTDIR

char *trayectoria no es una trayectoria correcta
