10.10.- Desarrollo del programa articulo.exe usando estructuras de datos                                            



Podemos imaginarnos el procesado de los articulos de una tienda de ultramarinos. Para ello partiremos de una estructura sencilla

{
  char NombreArticulo[10];

El programa debería de ser algo paracido a lo que sigue:



  • Presentar menú de opciónes del programa.

  • Introducir datos en el archivo.

  • Ordenar los datos del archivo.

  • Leer los datos que nos interesen del archivo.

Para ir eliminando errores, a medida que vayamos terminando las diferentes funciones de las que consta el programa iremos recomplilándolo para eliminar largas listas de errores. En principio vamos a desarrollar una función que nos permita tomar una opción determinada. Sólo figuraremos 3 opciones: Introducir datos, leer datos y salir del programa. Si necesitáramos añadir alguna opción más lo haremos en la medida que lo necesitemos. Como para poder probar el programa, necesitamos una función main(), también la desarrollaremos. Como primera prueba, desarrollaremos las diferentes funciones en diferentes ficheros, de manera que si el código se extiende quede más accesible, modular y nos resulte todo más fácil a fin de reconocer las diferentes funciones y errores. Decidimos poner como nombres al fichero el nombre de las funciones que contienen a fin de que nos resulte más fácil acceder a las diferentes funciones. Si observamos la sinopsis de fwrite(), veremos que el primer argumento que se pasa es la dirección de memoria de la varible que queremos escribir en el archivo. Por ejemplo si quisieramos escribir en el archivo "archivo.dat" el entero int numero, el primer argumento de fwrite() ha de ser &numero (la direcciónde la varible número). Podríamos pensar que el primer arguemento de fwrite() en la linea 30 sería &pun_art, pero sería un error debido a que punt_art es por sí solo una dirección, un puntero a un estructura. Hemos de observar que si quisiéramos escribir en el fichero una cadena de caracteres mediante fwrite(), tampoco habria que especificar el operador '&' delante del nombre de variable de la cadena debido a que el nombre de la variable de una cadena de caracteres es por sí solo una dirección de memoria.


El segundo argumento de fwrite() es sizeof(struct articulo). La función fwrite() escribirá el número de bytes especificados a partir de la dirección de memoria especificada por punt_art.

El tercer argumento, '1', simplemente especifica el número de bloques de datos que qieremos escribir. Si por ejemplo especificáramos un '2' escribiría 2 bloques de datos en el fichero apuntado por stream, escribiría dos veces la estructura apuntada por punt_art en el archivo apuntado por stream.


El cuarto argumento es el nombre del puntero FILE que apunta al fichero que nos interesa.
