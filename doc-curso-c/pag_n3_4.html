3.4.-Tipos de datos                                                 



Los diferentes tipos de datos en C ocupan diferentes longitudes de memoria, al mismo tiempo debemos de especificar el tipo de dato que vamos a asignar a una variable debido a que en el espacio de memoria destinado a ello sólo vamos a poder almacenar datos del tipo especificado. Si hemos declarado una variable de tipo char, el intento de asignación a dicha variable de un tipo float provocará un error técnico o de diseño que en un momento dado podría ser dificil de detectar.

Debemos de distinguir entre errores de compilación, falta de sintaxis, no permiten coimpilar. Errores de ejecución provovados por un desbordamiento del programa o bucles infinitos, permiten compilar pero en algún momento del programa este deja de ejecutarse. Errores de asignación, provocados por un deficiente desarrollo del programa. Este tipo de errores también permiten compilar, pero son errores muy dificiles de encontrar debido a que el programa arroja resultados, en este caso erroneos.



El tipo int y el operador sizeof()



Una variable de tipo int es una variable que contiene un número entero. Así mismo, el operador sizeof() devuelve el tamaño en bytes del tipo de variable o estructura (tipo de variable compuesta por diferentes tipos de variables y tamaño no concretado previamente) contenida en los paréntesis situados a continuación de sizeof. Ya explicamos antes cómo en diferentes plataformas una variable tipo int podría ocupar 4 bytes en el caso de Linux o 2 bytes en caso de Windows98. Para cercionarmos de estos pequeños inclonvenientes, C dispone de la función sizeof.

tamano = sizeof (float);



Esta sentencia asignará el el número de bytes de que consta una variable de tipo float en el sistema sobre el que se ejecuta. En este caso devolverá el número 4, que lo asignará a la variable tamamo.

Dependiendo de el sistema operativo que usemos y si nuestro procesador lo admite, el tipo int puede usar 2 o 4 bytes. Para saber cuantos bytes usa nuestra máquina para un tipo int, basta ejecutar un programa como el que sigue:


#include <stdio.h>

main()

{
   int tam_int;
   tam_int = sizeof(int);

   printf ("El tamaño de un int en mi sistema es de %d bytes\n", tam_int);

}



La sentencia sizeof() devuelve el valor en bytes de la variable que haya en el interior de los paréntesis.

Cuándo ejecutemos esta sentencia, obtendremos uno de los siguientes resultados:

El tamaño de un int en nuestro sistema es de 2 bytes


o bien


El tamaño de un int en nuestro sistema es de 4 bytes


Para que nos hagamos una idea, en una misma máquina se han obtenido ambos resultados. La explicación está en que los 2 bytes se han obtenido bajo el sistema operativo Windows9x. Sin embargo, se ha obtenido el valor de 4 bytes para el sistema operativo Linux, ejecutado en la misma máquina.

En que nos afecta esto? Muy sencillo, si nuestro sistema nos da un valor para un int de 2 bytes significa que cuando asignemos un número a una variable int, este no podrá exceder de 32767 ni ser inferior a -32768, límites que nos proporciona el rango de 2 bytes. Es fundamental conocer el tamaño de nuestros int, ya que eso nos permitirá conocer hasta qué límite podemos llegar en el sistema que usemos con las variables de tipo int.

El tipo long

Este tipo de variable, se usará en los casos que nuestras variables superen los límites del tipo int. Aunque si realmente deseamos que nuestro programa sea portable deberemos de declara siempre nuestra variable como long independientemente de la plataforma en la que estemos.

Si ejecutáramos una linea como esta:


valor_long=sizeof (long);


obtendríamos el valor 4 para la variable valor_long.

Overflow.- Si asignáramos en un sistema de 2 bytes (Windows 9x) para un int un valor superior a 32767, por ejemplo 32768 probablemente al compilar funcione, de hecho funciona (podria darnos algun mensaje de aviso warning), pero en la ejecución del programa se producirá un error de asignación, ya que tomará el valor -32768 en lugar de 32767. Cuando esto ocurre, se dice que ha ocurrido un caso de overflow. Esto es indiferente, el asunto es que no asigna lo que queremos donde queremos, pero ya sabemos como se comporta nuestro compilador. Es algo así como una rueda de ruleta en la que los números están ordenados secuencialmente y en la que son vecinos los límites superior e inferior. Cuando rebasemos los límites, el compilador comenzará por el valor inmediato. En el caso de un int de 2 bytes, serían vecinos los números -32768 y 32767, precisamente los límites del rango int. Si intentáramos asignar una variable int a un número superior a 32767, por ejemplo 32768 obtendríamos  el número -32768 con el consiguiente desastre para nuestro programa. Si tu sistema es Linux, podrás hacer esta comprobación con una variable de tipo short, que tiene los límites de de un int en un sistema de 2 bytes.

Comprueba el siguiente programa. Si usas Windows9x, debes de cambiar los tipos short a y short b por int a e int b:
 

/***********overflow2.c**************/


#include <stdio.h>

main()

{

   short a = 32768;

   short b= -32769;


   printf("a=%d, a+1= %d, a + 3= %d\n", a, a+1, a + 3);

   printf("b=%d, b-1= %d, b - 3= %d\n", b, b-1, b + 3);

}



a=-32768, a+1= -32767, a + 3= -32765

b=32767, b-1= 32766, b - 3= 32770


Hemos de fijarnos que a es declarada como un valor positivo y b los es como un valor negativo, sin embargo obtenemos resultados con signos opuestos.

El tipo char

Realmente el tipo de variable char almacena enteros con el rango -128 al 127. La variable unsigned char almacena los números enteros con el rango 0 a 255. Estos números se corresponden con los 256 caracteres ASCII. Evidentemente dentro de estos valores están comprendidos las letras y números de nuestro alfabeto. El rango de 0 a 127 contiene las letras y números de nuestro alfabeto así como la mayoría de los símbolos del teclado. Observemos los siguientes ejemplos:


char a = 'j';

char c = 23;


Sería idéntico que escribir lo siguiente , ya que 'j' se corresponde con el valor ASCII nº 106 y el carácter nº 23 con el símbolo '#':


char a = 106;

char c= '#';


Veamos el siguiente programa:


#include <stdio.h>

main()

{
  char a = 'a';
  char b = 'b';
  char c = 'c';
  char d = 'd';
  char gunyo='~';
  char almohadilla= '#';

  printf ("El valor ASCII de %c es %d\n", a, a);
  printf ("El valor ASCII de %c es %d\n", b, b);
  printf ("El valor ASCII de %c es %d\n", c, c);
  printf ("El valor ASCII de %c es %d\n", d, d);
  printf ("El valor ASCII de %c es %d\n", gunyo, gunyo);
}


El resultado de este programa es:

El valor ASCII de a es 97

El valor ASCII de b es 98

El valor ASCII de c es 99

El valor ASCII de d es 100

El valor ASCII de ~ es 126



En todo caso podrías comprobarlo con la tecla ALT + numero ASCII. Por ejemplo, el valor ASCII de '#' es 35. Si pulsamos la tecla ALT+35 (tecla 'ALT' y sin soltar la misma pulsar el número 35 y a continuación soltar la tecla 'ALT'), obtendremos el valor '#' escrito sobre nuestra pantalla de texto.

Observa las siguientes declaraciones:

latan = F; /*No es correcta. Debemos encerrar nuestro carácter a asignar entre comillas*/

Casting



No siempre es necesario hacer coincidir el tipo de dato asignado con el tipo de variable. Es contraproducente asignar tipos de datos de gran precisión a otros de menor precisión. Si asignamos un dato float a una variable de tipo int, la mayoría de los compiladores asignarán el dato pero eliminando la parte derecha de la coma. La forma de hacer esta conversión sería implementando un casting. Usariamos un casting para pasar un valor float a uno de tipo int de la siguiente manera:



float f = 4.087;

int num;

num = (int) f; //Este es el casting





En estas lineas de código, se asignará el valor 4 a int num, despreciandose la parte decimal en la operación de casting. En todo caso a la hora de hacer un casting hemos de tener muy en cuanta los límites de la variable a la que es asignada. Si en estas lineas intentáramos hacer una asignación int num de un valor superior a 2147483647, ocurriría que el valor superaría los límites designados para un int e incurriríamosen un nuevo caso de overflow. Cuando realicemos una operación de casting hemos de tener muy en cuenta tanto del valor que es asignado, como los limites de la variable que recibe la asignación.
