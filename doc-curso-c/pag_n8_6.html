8.6.- Punteros que apuntan a a arrays de estructuras


Tambien podemos hacer que el puntero a una estructura apunte a los diferentes elementos de un array de estructuras. La manera de hacerlo es símplemente sumando unidades al puntero dependiendo del elemento del array al que queramos acceder o bien asiganar al puntero la dirección de memoria del elemento del array que queramos leer:
 


#include <stdio.h>

main()

        { 

          int i;

          struct miestructura

                    {

                     int n_ladrillos;

                     char t_construct[20];

                     int coste;

                    };
 
 

           struct miestructura estr[2]=

                   {

                       /*Definimos estr[0]*/

                         {

                          1000,

                          "chabola y muro",

                          120000

                         },

                       /*Definomos estr[1] */

                         {

                          500,

                          "Torreón",

                          30000

                         }

                    };
 

            struct miestructura *punt=NULL;

/*asignamos la dirección de memoria del primer elemento del array en el puntero*/

            punt = &estr[0];

            for(i=0; i<2; i++)

                 {

                  printf("\nValores de estr[%d]\n", i);

                  printf("punt->n_ladrillos = %d\n", punt->n_ladrillos);

                  printf("punt->t_construct = %s\n", punt->t_construct);

                  printf("punt->coste = %d\n", punt->coste);

                  punt++;

                }

}





Es importante que nos fijemos en la posición de las llaves, comas y puntos y comas de la declaración del array de estructuras. La declaración strcuct miestructura estr[2], ha de disponer de dos llaves, quye a su vez contendrán tantas parejas de llaves como elementos disponga el array. Estas parejas de llaves han de estar separadas por comas. Básicamente la situción es análoga a la declaración de arrays cadenas de caracteres o cualquier otro tipo de estructura, la diferencia está en el uso de las llaves que en nuestro caso cumplen la función de agrupar los valores de las diferentes estructuras de que consta el array. Y te reclacamos que es importante que te fijes debido a que la falta de cualquiera de estas llaves o comas o puntos y comas originará un error de compilación.

Para que *punt apunte al primer elemento del array tansólo hemos de hacer la siguiente declaración:
 

punt = &estr[0];



Entramos en el bucle for() y leemos los datos de estr[0]. A continuación nuestro interés es que *punt disponga de la dirección del siguiente elemento del array, por tanto sólo hemos de sumar una unidad a punt mediante la sentencia
 

punt++;



Con lo que conseguimos que punt "avance" tantas unidades de memoria como unidades mide una estructura del tipo miestructura. Tambien podíamos haber desarrolado el programa si hacer la asignacion exterior al bucle for() de punt = &estr[0]; implementando todo el proceso en el bucle for() de la siguiente manera:
   
 


for(i=0; i<2; i++)

         {


           punt = &estr[i];

           printf("\nValores de estr[%d]\n", i);

           printf("punt->n_ladrillos = %d\n", punt->n_ladrillos);

           printf("punt->t_construct = %s\n", punt->t_construct);

           printf("punt->coste = %d\n", punt->coste);

           punt++;

}



Que como verás es algo muy similar a la solución anteriormente propuesta.
Otro forma de acceder a los elementos de un array de estructuras mediante un puntero es usando el operador de indirección '*' combinado con el operador de pertenencia '.' para referirnos a los diferentes elementos del array. En el programa antrior sería algo así como lo que sigue:
 


punt = &estr[0];

 
for(i=0; i<2; i++)

             {

              printf("\nValores de estr[%d]\n", i);

              printf("punt->n_ladrillos = %d\n", (*punt).n_ladrillos);

              printf("punt->t_construct = %s\n", (*punt).t_construct);

              printf("punt->coste = %d\n", (*punt).coste);

              punt++;

}





Si hemos declarado que punt = &estr[0], forzosamewnte si usamos el operador de pertenencia se cumplirá que
*punt == estr[0]; y en cualquier iteración de for() que nos refiramos por ejemploa a n_ladrillos se cumplirá la igualdad:

estr[0].n_ladrillos == (*punt).n_ladrillos

Y por similitud con lo visto hasta ahora, observarmos tambien que la siguiende igualdad se cumple:

estr[0].n_ladrillos == (*punt).n_ladrillos == punt->n_ladrillos

La razón por la que encerramos a punt entre paréntesis es que el operador de pertenencia '.' tiene preferencia sobre el operador de indirección '*' , y como queremos que la sentencia ejecute en primer lugasr lugar el operador de indirección, encerramos entre paréntesis a punt.El operador '*' actúa sobre una dirección de memoria que en esta caso es punt. Si no encerráramos entre paréntesis a punt, ocurriría que el operador de indirección actuaría sobre punt.n_ladrillos y punt.n_ladrillos no es una dirección de memoria. 

Con esto no pretendemos complicar al lector en el aprendizaje de punteros ni nada parecido, se trata de entender las diferentes notaciones de los punteros y que ciertos programadores suelen usar dichas notaciones de forma importante y es fundamental que cuando vemaos algo de esto no nos asustemos y sepamos que es de lo que se trata. Tal vez esta última notación. Sea la más complicada de comprender, pero ten encuenta que tan sólo se trata de diferentes mecanismos que nos llevan al mismo resultado. ¿Cuál escoger? ¿Cuál usar? Realmente el que mejor comprendamos será el mejor. La salida del programa en cualquiera de las sintaxis que hemos visto será:
 


Valores de estr[0]

punt->n_ladrillos = 1000

punt->t_construct = chabola y muro

punt->coste = 120000


 
 

Valores de estr[1]

punt->n_ladrillos = 500

punt->t_construct = Torreón

punt->coste = 30000
