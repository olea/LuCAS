13.1.- ¿Que es el preprocesado?



Cuando invocamos al compilador de C mediante la orden 'gcc archivo.c' o 'cc archivo.c', todo el proceso de compilación, podríamos dividirlo en tres etapas:


  • Preprocesado

  • Compilación

  • Enlace



En este capítulo nos vamos a ocupar del preprocesado, etapa que se encarga de dejar un archivo con código en C a punto para la compilación propiamente dicha, para ello elimina los comentarios y hace las sustituciones necesarias de las constantes declaradas en el exterior de la funciones. El preprocesado lee en primer lugar las lineas que comienzan por una almohadilla '#' y hace las sustituciones necesarias o incluye los archivos necesarios. Observemos el siguiente programa:

/* preprocesado1.c */


#include <stdio.h>



#define SI 1

#define NO 0

#define QUIZAS 2

#define NUMERO 7

main()

{
  //Aquí comienza el programa

  if (SI == 1)

       printf("SI == 1\n");


  if (SI != NO)

       printf("SI != NO\n");


  /*Estamos hacia la mitad del programa */

  if(QUIZAS != 3000)

printf("Tal vez QUIZAS sea diferente de 3000\n");


  printf ("NUMERO = %d\n", NUMERO);

}
//aquí termina el programa



Observamos que el programa disponde de varios comentarios:

/* preprocesado1.c */

//Aquí comienza el programa

/*Estamos hacia la mitad del programa */

//aquí termina el programa



Este programa tambien contine varias direcitivas de procesador:


#include <stdio.h>



#define SI 1

#define NO 0

#define QUIZAS 2

#define NUMERO 7



Es menester del compilador, que antes de compilar esté el archivo listo para ello, entonces antes de compilar ha de llamar al preprocesado. Si bien todo este proceso se hace con la única orden de 'gcc' o 'cc', podemos observar como queda nuestro programa una vez preprocesado mediante la orden:

gcc -E preprocesado1.c


Si ejecutamos esta orden en la consola, podremos ver algo parecido a lo siguiente:

# 1 "prepro1.c" 2


main()

{

if (1 == 1)

printf("SI == 1\n");


if (1 != 0 )

printf("SI != NO\n");



if(2 != 3000)

printf("Tal vez QUIZAS sea diferente de 3000\n");


printf ("NUMERO = %d\n", 7 );

}



Lo que hemos hecho ha sido preprocesar el programa preprocesado1.c sin compilarlo. Observa que esta orden ha eliminado los comentarios y ha sustituido las constantes SI, NO, QUIZAS y NUMERO por sus valores numéricos equivalentes designados en las directivas del preprocesador. Puedes retroceder en la consola (shift + Repág) para ver que más ha hecho el preprocesado y podrás ver que hay muchísimas líneas hacia atrás, no las podrás ver todas, porque esta orden tiene un límite de paginado, pero has de saber que son las líneas resultantes de la directiva '#include <stdio.h>'. Estas líneas no las incluimos aquí por razones de espacio, pero puedes observarlas completamente si ejecutas la orden:

gcc -E preprocesado1.c > preprocesado.txt



Esto generará un fichero llamado preprocesado.txt que podrás ver, examinar y comparar con stdio.h. Observa que las últimas 16 lineas de preprocesado.txt son las que antes vimos por pantalla.

Como ya sabemos que es lo que hace el preprocesador de C vamos a ver que directrices conoce:

#define

#elif

#else

#endif

#error

#if

#ifndef

#ifndef

#include

#line

#pragma

#undef



Si bien en un sistema Linux normalmente no debes de tener problemas donde colocar el símbolo almohadilla '#', por razones de compatibilidad con otros sistemas es muy importante situarlo al principio de la linea, escribiendo a continuación el nombre de la directriz, de hecho hay algunos autores que consideran un solo conjunto la almohadilla más la directriz, lo cual es muy acertado debido a la compatibilidad. Por ejemplo el compilador del sistema operativo HP-UX no reconoce una directriz de preprocesado colocada despues del primer carácter de la linea.
