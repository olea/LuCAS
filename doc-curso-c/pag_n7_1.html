7.1.- Paso de parámetros a una función



Supongamos que queremos intercambiar los valores de dos variables de un programa. Sería sencillo:





#include <stdio.h>
main()
  {
   int x = 10;
   int y = 51;
   int aux;

   printf("x = %d\n", x);
   printf("y = %d\n", y);
   aux = x;
   x = y;
   y = aux;
   printf("\nAhora hemos intercambiado los valores de x e y \n");
   printf("x = %d\n", x);
   printf("y = %d\n", y);
  }


Este programa es claro. Símplemente cambia los valores de las variables x e y mediante una variable de apoyo aux. La salida de este programa será:

X = 10
y = 51

Ahora hemos intercambiado los valores de x e y
x = 51
y = 10



Si intentáramos intercambiar los valores x e y mediante una función a primera vista haríamos algo parecido a esto:


/*******cambiador.c*********/
#include <stdio.h>
void intercambia(int x, int y);

main()
  {
   int x = 10;
   int y = 51;
   int aux;

   printf("x = %d\n", x);
   printf("y = %d\n", y);
   intercambia (x, y);
   printf("\nAhora hemos intercambiado los valores de x e y \n");
   printf("x = %d\n", x);
   printf("y = %d\n", y);
  }

void intercambia (int num1, int num2)
  {
    int aux;

    aux = num1;
    num1 = num2;
    num2 = aux;
  }


Este programa es correcto, de hecho no nos dará problemas de compilación y además funcionará. Lamentablemente no cumplirá los objetivos. Veamos el resultado:



x = 10
y = 51

Ahora hemos intercambiado los valores de x e y
x = 10
y = 51




¿Que ha pasado? Ocurre que a la función intercambia() le hemos pasado los valores x e y despues los intercambia, lamentablemente le hemos pasado una copia de los mismos, no los valores verdaderos. Las funciones en C son muy desconfiadas y cuando envían valores a otra función le envía un copia de los mismos, es decir se crean otras variables con idéntico valor pero con distinta dirección de memoria con el fin de que dichos valores no puedan ser alterados incontroladamente. Cuando termina de ejecutarse la función invocada estos valores desaparecen y de hecho en la mayoría de las ocasiones se pierden. El asunto es que para poder intercambiar dichos valores no debemos de enviar los valores de las variables a la función invocada, sino las direcciones de memoria de las variables que queremos intercambiar. Y ya estamos de nuevo en el terreno de los punteros. Los operadores que rigen la sintaxis de los punteros son el operador de dirección '&' y el de indirección '*'. Cuando anteponemos el operador & a una variable estamos refiriendonos a su dirección de memoria y cuando anteponemos el simbolo '*' la misma variable nos estamos refiriendo al contenido de la dirección a la que apunta. Veamos:

Cuando declaramos la variable int *punt = NULL estamos declarando que punt es una variable puntero que sólo va a aceptar como valor una dirección de memoria que apunte a un entero. En el momento de la declaración de las variables punt contiene NULL, esta inicializada al valor nulo NULL. Es buena costumbre inicializar los punteros apuntando a NULL de esta forma. Tambien podriamos declararlo de la forma int *punt. La diferencia redicaría en que de esta forma punt contendría basura.




Cualquier variable puntero ocupa igual espacio que un entero, pues es espacio suficiente para almacenar este tipo de datos de direcciones de memoria. Por otro lado hemos inicializado una variable int numero con un valor de 13. En esta declaración se crea una dirección de memoria que contiene el valor 13 y para referirnos a esta dirección de memoria y a su contenido (13) nos basta con invocar la cadena numero. Si quisiéramos conocer la dirección de numero, bastaría con anteponer el símbolo '&' a numero de la forma &numero. Esto nos devolvería la dirección de memoria de la variable numero.
Como punt sólo acepta direcciones de memoria como valores sería correcto realizar la siguiente sentencia:



8

   punt = &numero;




De esta manera punt ya no apuntaría a NULL, sino a la dirección de memoria de la variable numero. En este preciso instante decimos que punt apunta a numero. La salida del programa sería:




punt = (null)
Ahora inicializamos punt.
numero = 13
punt = 2000037
*punt = 13
&numero = 2000037
&punt = 5000321




Observamos las semejanzas de los valores de las variables:

numero == *punt

Cuando especificamos *punt estamos refiriéndonos al contenido de la dirección de memoria a la que apunta punt. punt tiene alamcenada la dirección de memoria de numero. *punt y numero han de referirse al mismo valor.


Cuando especificamos punt estamos refiriéndonos a un dirección de memoria que puede contener la dirección de número o la de otra variable entera cualquiera.

punt == &numero



Con estos conocimientos ya estamos en condiciones de intercambiar los valores de cambiador.c, ya que el problema lo teníamos en que enviábamos como parámetros los valores de las variables a la función intercambia(). Ahora ya estamos enterados que debemos de enviar sus direcciones de memoria:





/*******cambiador.c*********/

#include <stdio.h>

void intercambia(int x, int y);


main()
  {
   int x = 10;
   int y = 51;
   int aux;

   printf("x = %d\n", x);
   printf("y = %d\n", y);
   intercambia (&x, &y);
   printf("\nAhora hemos intercambiado los valores de x e y \n");
   printf("x = %d\n", x);
   printf("y = %d\n", y);
  }

void intercambia (int *num1, int *num2)
  {
   int aux;

   aux = *num1;
   *num1 = *num2;
   *num2 = aux;
  }



La salida de este programa es:




x = 10
y = 51

Ahora hemos intercambiado los valores de x e y
x = 51
y = 10


El programa funciona perfectamente. En este programa intercambia() recibe como parámetros una copia de las las direcciones de memoria de x e y. Dichas copias son declaradas de la forma int *num1 e int *num2. Esto nos permita afirmar que:

num1 == &x
num2 == &y

y que

*num1 == x
*num2 == y

Ahora sólo hemos de intercabiar los valores apuntados por num1 y num2, ya que estamos trabajando sobre la dirección de memoria que nos interesa cambiar que es precisamente la direccion de memoria de x e y.
