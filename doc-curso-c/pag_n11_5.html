11.5.- Cambio del usuario propietario



Tambien podemos cambiar a los propietarios y grupos de los archivos con las siguientes funciones:



Sinopsis de chown() y fchown()

#include <sys/types.h>
#include <unistd.h>

int chown(char *trayectoria, uid_t owner, gid_t group);

int fchown(int descriptor, uid_t owner, gid_t group);

Devuelve 0 en caso de éxito y -1 caso de error




Cambia el propietario del fichero especificado por char *trayectoria ó int descriptor. Solamente el superusuario puede cambiar el propietario de un fichero. El propietario de un fichero puede cambiar el grupo de dicho fichero a cualquier grupo al que dicho usuario es miembro. El superusuario puede cambiar el grupo arbitrariamente.


Los errores más generales para chown() son:




Código de error

Descripción

EPERM

El UID efectivo no coincide con el propietario del fichero, y no es cero: ó el propietario (owner) o grupo (group) han sido especificados incorrectamente.

EROFS

El fichero en cuestión reside en una zona de solo-lectura del sistema de ficheros.

EFAULT

char trayectoria señala fuera de su espacio de direcciones accesible.

ENAMETOOLONG

char *trayectoria es demasiado largo.

ENOENT

El fichero no existe.

ENOMEM

No hubo suficiente memoria en el kernel.

ENOTDIR

Un componente del prefijo char *trayectoria no es un directorio.

EACCES

Permiso de busqueda denegado en un componente de char *trayectoria

ELOOP

char *trayectoria contiene una referencia circular -por ejemplo, una ligadura simbólica a si misma-.
