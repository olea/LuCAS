7.3.- Funciones para el manejo de cadenas de caracteres



Para poder comparar o unir cadenas, buscar caracteres en ellas y otras muchas aplicaciones, C dispone de varias funciones desarrolladas especificamente para estos menesteres. Existen momentos en los que usar sizeof() se hace casi imposible o demasiado tedioso para ello usaremos una función llamada strlen(). Otros momentos queremos copiar una cadena en otra y deshechar la última, tambien podemos querer ordenar cadenas por orden alfabético, etc, etc, etc. Veamos como se usan y manejan las más interesantes. Hemos ejecutado la orden man <función buscada> en la consola de Linux y por supuesto hemos obtenido bastante información sobre la función buscada. En primer lugar de cada explicación de función figuraremos el modo de usar dicha función tal como nos aparece con dicho comando. Incluiremos funciones que sirvan para compiladores Borland, Microsoft o gcc contempladas por ANSI C. A continuación echaremos un vistazo a varias de las funciones más usadas para manipular cadenas de caracteres.


Función strlen()


NOMBRE strlen - calcula la longitud de una cadena de caracteres excluyendo el carácter nulo de treminación de cadena\0

SINOPSIS
#include <string.h>
size_t strlen(char *cadena)

Compatibilidad: ANSI, UNIX-Linux, MSDOS


Esta función ha de devolver un número entero. Sin embargo en vez de aparecer como int strlen(char *cadena) se muestra como size_t strlen(char *cadena) . size_t  está definido en el fichero stdio.h como un valor unsigned int lo cuál es bastante acertado pues ninguna cadena de caracteres dispone de una longitud negativa. Como tema un tanto asustante está el contenido de los paréntesis. Esto símplemente significa que aquí ha de situarse un puntero a char, un puntero a uno o varios caracteres. Como el nombre de un array de caracteres es un puntero a una cadena de caracteres ya lo tenemos resuelto: Hemos de tener en cuenta que esta función a la hora de hacer mediciones no contempla el caracter '\0'.


#include <stdio.h>
#include <string.h>

main()
{
  char saludo[] = "hola";

    printf("saludo medido con sizeof mide %d", sizeof(saludo));
   printf("saludo medido con strlen mide %d", strlen(saludo);
}

Funciones strcat() y strncat()




NOMBRE
strcat - Añade cadena_2 a cadena_1. Incorpora el carácter nulo a la cadena resultante devolviendo un puntero a cadena_1.
strncat - Añade los primeros n primeros caracteres de cadena_2 a cadena_1 y devuelve un puntero a cadena_1 .

SINOPSIS
#include <string.h>
char *strcat(char *cadena_1, char *cadena_2);
char *strncat(char *cadena_1, char *cadena_2, size_t n);

Compatibilidad: ANSI, UNIX-Linux, MSDOS



Para que puedas usar esta función has de incluir el archivo de cabecera <string.h>. strcat() desacarta el carácter nulo de cadena 1, añade o concatena cadena_2 con cadena_1 y termina la cadena resultante con el carácter nulo \0. Devuelve un puntero a la cadena resultante. Observa el siguiente programa:

#include <stdio.h>
#include <string.h>

{
  char saludo[]="Buenos días ";

  char *p;

  printf("Como te llamas? ");
  gets(nombre);
  p = strcat(saludo, nombre);
  printf("Usando saludo = %s\n", saludo);
  printf("Usando p = %s\n", p);
}



Este programa pide un nombre al usuario y lo almacena en el array nombre[50] mediante la función gets(). Como la pretensión del programa es que una las cadenas saludo y nombre y muestre el resultado por pantalla, invocamos a la función strcat() y le pasamos como argumentos las cadenas saludo y nombre. Al mismo tiempo almacenamos el valor devuelto por strcat() en el puntero *p. En este momento tenemos dos punteros que apuntan a una misma dirección de memoria: p y nombre, de hecho al visualizarlos veremos que arrojan idéntica cadena de caracteres. Funciones strchr() y strrchr()


NOMBRE strchr - Busca el primer carácter c en cadena.
strrchr - Busca el último carácter c en cadena.
SINOPSIS
#include <string.h>

char *strchr(char *cadena, int c);
char *strrchr(char *cadena, int c);

Compatibilidad: ANSI, UNIX-Linux, MSDOS



Esta función funciona insertando como argumentos la cadena donde queremos buscar el carácter y el carácter que queremos buscar. Si la cadena tuviera varios caracteres iguales al buscado, sólo podremos conocer la situación del primer carácter encontrado. Si se diera la circunstancia de que el carácter no estuviera en la cadena la función strchr() devolverá un valor NULL. Veamos como sucede esto y para que nos puede servir:

/*********************apellido.c************************/ #include <stdio.h>
#include <string.h>
main()
{
  char nombre_apellido[81];
  char *p;
  char apellido[20];

  gets(nombre_apellido);

     {
      if(*(p+i) != ' ')
         apellido[j++] = *(p+i);
      i++;
     }

  apellido[j] = '\0';
  printf("\np apunta a |%s|\n", p);
  printf("apellido apunta a |%s|\n", apellido);
}


Este programa pide el nombre y los apellidos del usuario y los almacena como una sola cadena de caracteres en el array nombre_apellido[81]. El programa trata de distinguir el apellido del usuario en dicha cadena y almacenarlo en el array apellido[20]. Como suponemos que el apellido y el nombre del usuario del programa están separados por al menos un espacio en blanco y que su nombre no es compuesto, localizaremos la dirección de memoria que tiene el primer espacio en blanco que aparece en la cadena nombre_apellido[]. Para ello usamos la función strchr() que recibe como argumentos la dirección de memoria el primer elemento del array, y el carácter que queremos encontrar en la cadena. La dirección de comienzo del array es precisamente el nombre del array y el carácter a buscar lo encerramos entre comillas simples. Como esta función devuelve la dirección de memoria del carácter buscado, la almacenaremos en el puntero a char p. Para almacenar el apellido en una cadena destinada a almacenar este tipo de datos, usamos a continuación un bucle while() y una sentencia if(). Este bucle funciona sólo cuando el carácter leido sea un final de cadena o carácter '\0'.

Observarás que para ello compara el dato al que apunta *(p+1) con el carácter nulo o de final de cadena. Si el dato al que apunta (p+1) no es fin de cadena se ejecuta la sentencia while(). Para que en la cadena apellido se pueda almacenar un carácter ha de darse el caso que el dato apuntado por p+i no sea el espacio en blanco, una pena pues aunque hay diversos apellidos con varios espacios en blanco el programa no contempla los apellidos compuestos. El bucle while() almacena en la cadena apellido[] todos los caracteres que hay desde el primer espacio en blanco de la cadena nombre_apellido hasta el carácter '\0', excepto los caracteres de espaciado. Si el usuario del programa se apellidara por ejemplo "Del Buen Suceso" en el array apellido se almacenará la tira: "DelBuenSuceso", que como verás carece de espacios en blanco. A continuación almacena el carácter nulo al fianl de la cadena apellido[]. Esto hemos de hacerlo "a mano", y sirve al programa para conocer dónde termina la cadena de caracteres. Hemos mostrado p y apellido a través de printf() y hemos encerrado su contenido entre barras para que se pueda advertir como elimina el carácter espaciado en la ejecución del programa. Por fín la salida del programa ha de ser algo parecido a lo siguiente:

Escribe tu nombre y apellidos: Luke Skywalker

p apunta a | Skywalker|
apellido apunta a |Skywalker|



La función strrchr() devuelve un puntero a la última aparición del carácter c en la cadena objetivo cadena. Si el carácter no es encontrado devuelve el valor NULL.

Veamos que ocurre cuando el carácter no es encontrado:

Funciones strcpy() y strncpy()


NOMBRE strcpy, strncpy - copian una cadena sobre otra si hubiera sitio para ello

SINOPSIS
#include <string.h>
char *strcpy(char *cadena_destino, char *cadena_fuente);
char *strncpy(char *cadena_destino, char *cadena_fuente size_t n);

COMPATIBILIDAD ANSI, UNIX-LINUX, MSDOS



La función strcpy() copia la cadena apuntada por cadena_fuente al array apuntado por cadena_destino incluyendo el carácter de terminación nulo ('\0') y devuelve un puntero a cadena_destino.
Se pueden cambiar dos cadenas de caracteres símplemente asignando la dirección de cadena_destino por la de cadena_fuente. Realmente esta función copia las cadenas de caracteres apuntados por cadena_fuente en cadena_destino. Los valores de los punteros a las cadenas cadena_fuente y cadena_destino mantienen el valor que tenían antes de ejecutarse la función strcpy(). Podría darse el caso que el tamaño de cadena_destino fuera más pequeño que cadena_fuente con lo que strcpy() comenzaría a escribir los caracteres sobrantes de cadena_fuente en lugares que no son apropiados. Esto puede originar cualquier cosa inesperada. Comentan de un programador que no contempló esta circunstancia y su monitor comenzó a saltar por la habitación por tanto os aconsejo que antes de ejecutar este tipo de funciones comprobeis el tamaño de cadena_destino. Las páginas man de Linux no dudan en calificar a un programador que no contemple esta circunstancia como stupid/lazy (tonto/perezoso).

La función strncpy() es similar a la anterior excepto que sólo copia en cadena_destino el número de bytes o caracteres especificados en size_t n. Esta función no añade el carácter nulo (\0) al terminar de copiarse en cadena_destino . Si se diera el caso que cadena_fuente fuera menor que size_t n el espacio restante será completado con caracteres nulos. Observa el elemplo:

#include <stdio.h>
#include <string.h>

main()
{
  char saludo[12]= "Buenos dias";
  char dos_letras[] = "to";
  int i;

  strncpy(saludo, dos_letras, 6);
  for(i=0; i<12; i++)
     {
      if(saludo[i]=='\0')
          {
            putchar('0');
            continue;
          }
       putchar(saludo[i]);
     }
  puts("\n");
}


Este programa copia 6 caracteres de dos_letras en la cadena saludo. Como dos_letras tiene sólo 3 caracteres o tres bytes ('t', 'o', '\0') el espacio restante lo rellena con caracteres nulos en la cadena saludo. Para poder visualizar los caracteres nulos hemos desarrolado un bucle while() con un if() anidado que cambia los caracteres nulos por caracteres '0'. Veamos la salida del programa:

to0000 dias0


El resultado del programa sustituye por '0' todos los caracteres nueva línea, incluso el de terminación de la cadena saludo



Funcion strcmp() y strncmp()


NOMBRE strcmp, strncmp - Comparan dos cadenas de caracteres, distinguiendo entre mayúsculas y minúsculas. Devuelve un valor indicativo del resultado:

< 0 si cadena1 es menor a cadena2
= 0 si cadena1 idéntica a cadena2
> 0 si cadena1 es mayor a cadena2

SINOPSIS
#include <string.h>
int strcmp(char *cadena_1, char *cadena_2);
int strncmp(char *cadena_1, char *cadena_2, size_t n);

COMPATIBILIDAD UNIX, MSDOS



Esta función traduce los caracteres de cadena_1 y cadena_2 a sus respectivos valores numéricos y luego compara ambas cadenas carácter por caracter. Si el resultado de la función es menor que 0 significa que cadena_1 está situada alfabéticamente antes que cadena_2, si el valor que devuelve la función strcmp() es mayor que 0 la cadena_1 estará situada alfabéticamente despues que cadena_2. Si el valor devuelto por strcmp() es igual a 0 significa que ámbas cadenas son iguales. Funciones strspn() y strcspn()


NOMBRE

strcspn.- Devuelve la posición del primer carácter de cadena_1 contenido en cadena_2.
SINOPSIS
#include <string.h>
size_t strspn(char *cadena_1, char *cadena_2);
size_t strcspn(char *cadena_1, char *cadena_2);

COMPATIBILIDAD ANSI, UNIX-LINUX, MSDOS





Si ningún carácter de cadena_2 está contenida en cadena_1 la función devuelve la posición del carácter de terminación de cadena (carácter nulo ó '\0') en el array cadena_1.

Fíjate que programa. Aparentemente ha de funcionar bien. El programa pide un entero por el teclado, se lo damos y lo mostramos por pantalla. Vamos a contemplar que el usuario de nuestro programa en el momento de pulsar un número entero para nuestro programa a la altura de la segunda cifra nota y advierte que un rodaballo de secano entra volando por su ventana regalando motocicletas amarillas. Si nuestro usuario del programa no tiene los nervios de acero, la probabilidad de que pulse una tecla no numérica aumentará considerablemente. Cuándo esto sucede nuestro programa interpretará cualquier cosa inesperada y aplicará valores inesperados a las variable para la que estamos introduciendo datos. Hemos de inventar alguna función que resuelva satisfactoriamente esta crítica situación, no podemos dejar a la buena intención o disposición del usuario del programa una labor de dias o meses. Deberíamos de desarrollar una función que analice carácter por carácter los caracteres inroducidos por teclado y cuando uno de ellos no sea un carácter no numérico, devolverá un mensaje de error en la captura de datos.

Una de las ayudas que nos proporciona C es la función atoi()


NOMBRE atoi ()- Convierte una tira de caracteres a un valor entero int.

SINOPSIS
#include <stdlib.h>
int atoi(char *cadena);
Compatibilidad: UNIX, MSDOS




NOMBRE atof() - Convierte una tira de caracteres a un valor float.

SINOPSIS
#include <stdlib.h>

float atof(char *cadena);
Compatibilidad: UNIX, MSDOS




NOMBRE atol()- Convierte una tira de caracteres a un valor long.

SINOPSIS
#include <stdlib.h>

long atol(char *cadena);
Compatibilidad: UNIX, MSDOS


Si pasamos una cadena de caracteres numéricos como parámetros a la función atoi(), esta devuelve el valor entero representado por los caracteres. En el caso de cadena contuviera algún carácter no numérico la función sólo leerá caracteres hasta el carácter mencionado. Veamos un ejemplo:

#include <stdio.h>
#include <stdlib.h>

main()
{
  char cadena[80];
  int = num;


  printf("Escribe un número para convertirlo a entero: ");
  gets(cadena);
  num = atoi (cadena);
  printf("cadena = %s\num = %d\n", cadena, num);
}
