10.12.- Acceso a los ficheros de forma no secuencial, fseek()



Sería muy tedioso tener que empezar siempre a leer un fichero por el principio. Para que esto no sea así, alguien inventó la función fseek(). Esta función mueve el puntero de L/E a la posición determinada por el programador. Si consultamos las páginas man de Linux tecleando man fseek(), obtendremos algo parecido a esto:


Sinopsis fseek()

#include <stdio.h>

int fseek(FILE *stream, long offset, int whence);

Esta función devuelve 0 en caso de éxito.


Donde long offset es el número de bytes que queremos avanzar o retrasar -aplicaríamos un valor negativo a offset- el puntero de L/E desde la posición dada por int whence. El meollo de la cuestión está en int whence. Este parámetro sirve para especificar el lugar desde el que queremos partir para avanzar o retrasar puestos de bytes por el fichero apuntado por stream. Cuando usemos esta función, int whence podrá tomar tres valores:


Nominación de int whence

Posición del puntero de L/E

SEEK_SET

Principio del fichero

SEEK_CUR

Posición del puntero de L/E en el momento de llamada a la función

SEEK_END

Final del fichero


Estos parámetros de fseek(), están definidos en stdio.h de la siguiente forma:


#define SEEK_SET 0 /* Seek from beginning of file. */

#define SEEK_CUR 1 /* Seek from current position. */

#define SEEK_END 2 /* Seek from end of file. */


Esto significa que si lo ves más claro puedes sustituir en el programa donde uses fseek() las palabras SEEK_SET, SEEK_CUR y SEEK_END por las los números 1, 2 ó 3 respectivamente.


Imaginemos que temos un fichero con 10 estructuras de 20 bytes cada una. Este fichero tiene un tamaño total de 200 bytes (10 estructuras * 20 bytes / estructuras). Para acceder a la 3ª estructura en el fichero, podríamos hacerlo de la siguiente manera:



FILE *stream;

fseek(stream, sizeof(struct exin) * 2, SEEK_SET);


Parece un tanto extraño que queramos avanzar tres puestos en el fichero, pero sin embargo multipliquemos el número de bytes por 2 en vez de hacerlo por tres. Vamos a imagianr que el cuadro siguiente es el fichero por el que nos estamos desplazando, la flecha de la izquierda es la posición desde donde queremos avanzar o retrasar el número de bytes especificaados en fseek (long offset) y los diferentes colores de la izquierda son las diferentes estructuras de 10 bytes cada una que contiene el fichero ejemplo. Tambien aparece el uso de fseek() para acceder a la posición de la estructura 3 del fichero ejemplo, bien contando desde el principio del fichero (SEEK_SET), bien desde el final de fichero (SEEK_END) :

Aplica este método cuando queramos escribir en la posición de la estructura 3, y tendremos que avanzar 20 bytes (sizeof(struct exin) * 2) desde la posición 0.
