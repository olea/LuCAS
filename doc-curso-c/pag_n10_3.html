10.3.- Función fopen()                               



Sinopsis fopen()

#include <stdio.h>
FILE *fopen(char *nombre_de_fichero, char *modo);

Compatible MSDOS, UNIX


Antes de manipular un archivo, es necesario abrirlo, para a continuación realizar sobre él las operaciones que necesitemos. Como podemos ver en la sinopsis, la función fopen() necesita como parámetros una cadena de caracteres para dar nombre al fichero y otra cadena que especifica el modo en que vamos a abrir el fichero.

Modo

Definición del modo

"r"

(r)ead. Abre un fichero para lectura. Si el fichero no existe obtenemos un error.

"w"

(w)rite. Abre un fichero para escritura. Si el fichero existe su contenido se destruye y si no existe se crea un fichero con el nombre esfecificado en la función fopen().

"a"

(a)ppend. Abre un fichero para escribir al final del mismo. Si el fichero no existe se crea

"r+"

Abre un fichero para leer y escribir. El fichero debe de existir.

"w+"

Abre un fichero para lectura y escritura. Si el fichero existe su contenido se destruye y si no existe se crea un fichero con el nombre especificado en la funcion fopen().

"a+"

Abre un fichero para leer y añadir datos al final del mismo. Si el fichero no existe se crea uno nuevo con el nombre especificado en la función fopen().


Si el programa va a correr sobre MSDOS es conveniente posponer al parámetro de modo de apertura del fichero el carácter 'b' o 't' cuando queramos abrir el fichero en modo binario o en modo texto respectivamente. La ausencia de estos identificadores implica que por defecto el archivo es abierto en modo texto. Afortunadamente para los usuarios de sistemas UNIX, este carácter es ignorado, no obstante no está de más especificarlo a fin de buscar la compatibilidad con otros sitemas operativos.

En un archivo de texto las operaciones de entrada y salida se conocen como "operaciones formateadas". Son archivos de texto y su lectura han de hacerse mediante funciones orientadas a caracteres o texto. Un archivo binario está formado por estructuras de datos y estas estructuras pueden tener campos que no sean caracteres. La información en estos archivos es y debe ser realizada en modo binario.

La función fopen() crea un flujo de información, canal o stream por el cuál podemos leer y escribier en fichero referenciado. Tambien crea un buffer de datos en la memoria en el que se almacena los datos insertados o borrados del fichero. Esta función devuelve un puntero al stream creado. Aunque esto parezca un poco fatigoso de comprender a la hora de usar dicha función, veremos que es muy manejable y fácil de usar.

Los errores que nos puede dar la apertura de un fichero mediante fopen() son varios:

  • Si se supera el número máximo de archivos en el sistema.

  • No tener permiso para abrir el archivo especificado

  • Haciendo mal uso de los identificadores de modo.

En caso de error, el puntero devuelto por la función fopen() será NULL, con lo que podemos gestionar perfectamente los errores cometidos.

Veamos como usar la función fopen() para abrir un fichero en modo lectura:

#include <stdio.h>
FILE *pf;

main()

    {
     char n_fichero[10];

     printf("Escribe un nombre para el fichero: ");
     gets(n_fichero); // Ya tenemos el nombre para dar al fichero
     pf = fopen(n_fichero, "w"); /* abrimos un fichero con el nombre contenido en n_fichero en modo escritura*/

     if(pf == NULL) /* Analizamos la no apertura del fichero para salir del programa si esto ocurre */
          {
           printf("Error: El fichero no puede abrirse\n");
           exit(1);
          }
     fclose(pf); //Cerramos el fichero
    }




Este programa no hace nada, simplemente abre un fichero en el direcrorio en el que ejecutamos el programa y a continuación cierra el stream asociado al fichero mediante fclose(). Puedes intentar compilar y ejecutar este programa, verás como en el directorio en el que lo ejecutes aparecerá un nuevo archivo llamado como lo has hecho en el programa.

Hemos cerrado el fichero con fclose(). Podríamos omitir esta linea y el programa se ejecutaria normalmente, de hecho al finalizar el programa se cierra automáticamente el fichero. Pero ocurre que el número de ficheros abiertos en cualquier sistema está limitado, por lo que aconsejamos ejecutar esta función despues de usar el fichero referenciado.
