1.4.- Un programa en C




Para desarrollar y compilar todos los programas de este manual, se desarrollará con el editor vim o xemacs y el compilador de Linux gcc. La distribución usada es SuSE (en este momento va por la versión 6.3). Esto no es obstáculo para poder escribirlos y compilarlos con Borland C, gcc bajo Windows 9x, Microsoft C o VisualC++.

Vamos a desarrollar un excelente programa. Un programa amable y bien educado que nos salude y que amablemente se retire cuando lo haga. Queremos que el programa nos salude por la pantalla. No queremos que haga nada más.

Pensemos en el programa. Básicamente este programa sólo necesita de la siguiente instrucción, que sería más o menos así:



orden

contenido de la orden

Mostrar en pantalla

"Hoy tienes muy buena cara, buenos dias"



¡Estupendo! Este programa es el colmo de la programación, es inteligente, sabe que nos puede aburrir, por eso saluda y se marcha. Vamos a ver cómo desarrollaríamos este estupendo programa en C:



/*******hola.c***************/
#include<stdio.h>
main()
{
  printf("Hoy tienes muy buena cara. Buenos días");
}
/****************************/



¿Que es esto?

Vamos a comentar este programa:

#include <stdio.h> Esto se llama directiva para el preprocesador, una tontería que puedes comentar en tu próxima fiesta y que queda muy bien. Cuando compilemos el programa, esta directiva #include le indicará al compilador que incluya el fichero stdio.h. En los sistemas UNIX este fichero está alojado en /usr/include/stdio.h. Podréis verlo con cualquiereditor de textos, no es un fichero binario. Este fichero como bien podreis deducir, contiene los parametros necesarios de entrada y salida de datos del programa. Algo así como standard in out. Absolutamnete todos los programas que admitan datos por teclado, o los expulsen por la salida estandard (pantalla, por ejemplo), necesitan incluir este fichero mediante esta directiva #include. Un 90% de los programas hechos en C necesitan de este fichero. Nosotros lo vamos a necesitar casi siempre.

Al estar stdio.h encerrado entre "< >" , significa que el fichero stdio.h ha de buscarlo en /usr/include/. Si fuera escrito #include "stdio.h", significaría que el compilador debería de buscarlo en el directorio donde este ejecutandose gcc.

main()  Los programas en C pueden contener muchas funciones o módulos diferentes, pero todos los programas han de tener una función main() o función principal. De esta función, derivarán todas las demás funciones y el programa en sí. El contenido o instrucciones de las funciones, en este caso main(), ha de ir siempre encerrado entre llaves {........} .

printf("Hoy tienes muy buena cara. Buenos días"); La función printf() expulsa por la pantalla el contenido de los paréntesis, en este caso el texto que hay. La función printf(), Es muy poderosa y puede imprimir los contenidos de todas las variables del programa. Pero esto lo dejamos para más adelante.

Tambien es de destacar, que todas las declaraciones y funciones terminan en ";". En el caso de que se nos olvide, obtendremos un error del compilador y no podremos generar el fichero ejecutable.
