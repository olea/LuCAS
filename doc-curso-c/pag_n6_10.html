6.10.- Arrays                                           



Tambien llamados arreglos. Un array es una serie de variables del mismo tipo. Pueden ser de cualquier tipo: int, float, char, etc.

Cuando definimos un array hemos de especificar el tipo de elementos que va a alojar en su interior, el nombre del array y la cantidad de elementos de que consta. Una vez definido el array, C reserva memoria para la totalidad del mismo de una forma contigua, de forma que podamos acceder a un cualquier elemento del array. Todos los elementos del array tienen direcciones de memoria consecutivas. Por ejemplo:
 

int erno[10]    /*Declara un array de 10 elementos de tipo int */
int turismo[5] = {1, 2, 3, 4, 5};


El primer elemnto de un array es el elemento nº 0. El primer elemento del array turismo es turismo[0] y el último es turismo[4]. El array turismo tiene 5 elementos y los números comprendidos entre 0 y 4 son 5, luego el array tiene 5 elementos. Los arrays están íntimamente ligados a la función for() y de vez en cuando a otras funciones como while(). Vamos a desarrollar un programilla que nos permita visualizar por pantalla los elementos de un array: Allá vamos:
 

#include <stdio.h>
main()
   {
    int numeros[5]={1, 2, 3, 4, 5};
    int i;

    for(i=0; i<5; i++)
          {
           printf("numeros[%d] = %d\n", i, numeros[i]);
          }
   }


La novedad en este programa es que cuando nos referimos a numeros[i] nos estamos refiriendo a una variable de tipo int que pertenece a una array de enteros llamado numeros y que ocupa el puesto i en el array.
La salida de este programa será:
 

numeros[0] = 1
numeros[1] = 2 
numeros[2] = 3
numeros[3] = 4
numeros[4] = 5



Observa la posición de array y el elemento que contiene. Para introducir elementos en el array procederemos como de costumbre con la función scanf() o incluso getchar() en el caso de que el array fuera una array de caracteres. Observa en el siguiente programa que lo primero que hacemos es declarar el array. Al mismo tiempo aprovecharemospara ver el contenido del array sin inicializar:

Hemos podido ver como se asignan eleemntos a un array. Ya hemos asignado 5 valores enteros a nuestro array, los hemos leido y visto. Los arrays de tipo static son inicializados a 0. En el siguiente programa didáctico, vamos a ver por pantalla el contenido de una array static y vamos a inicializar a 0 un array auto:




#include <stdio.h>

main()
   {
    static int arr_static[4];
    float arr_auto[3];
    int i;
    /*Visualizamos el array static*/
    for(i=0; i<4; i++)
          {
           printf("arr_static[%d] = %d\n", i, arr_static[i]);
          }
    for(i=0; i<3; i++)
          arr_auto[i] = 0;

    printf("\n\n");

    for(i=0; i<3; i++)
          {
           printf("arr_auto[%d] = %d\n", i, arr_auto[i]);
          }
   } 

 




En este programa no inicializamos el array static, pero si visualizamos su valor ,nos damos cuanta que todos sus elementos tienen el valor 0. A continuación inicializmaos todoslos elementos de arr_auto a 0, pues como vimos antes su cantenido era auténtica basura. Observa la salida del programa:




arr_static[0] = 0
arr_static[1] = 0
arr_static[2] = 0
arr_static[3] = 0

arr_auto[0] = 0
arr_auto[1] = 0 
arr_auto[2] = 0




En el caso de nuestro programa adverimos que arr_auto[] tiene tres elementos si quisieramos inicializar un elemento que no existiera a 0 el programa nos daría un error y no podría continuar con su ejcución.si arr_auto[3] tiene tres elementos, cuando intentemos hacer una cosa similar a:


 

arr_auto[7] = 0




se producirá un importante error, debido a que hemos invadido una parte de la memoria destinada a otro menester que seguro nada tiene que ver con arr_auto[]. Cuando declaramos la variable
int arr_auto[3] el compilador reserva un espario de memoria de tamaño:
  3 * tamaño en bytes de int, que es lo mismo que decir en lenguaje de programación :



3 * sizeof(int);



Cuando intentamos asignar un valor a un elemento de un array que está fuera de rango el compilador nos dará un error. Puede darse el caso que no te de error permitiendote compilar, pero el programa será incapaz de ejecutarse. Esta situación la denominarmos overflow. Las siguientes lineas son incorrectas:
 

int numeros[3];     /* Declaramos un array de 3 elementos */
numeros[4] = 17;    /* Incorrecto, asignamos un int a un elemento fuera del array */



Los arrays de caracteres nos permiten almacenar cadenas de caracteres, de hecho es la manera que tiene C de almacenarlas. Sin embargo las cadenas de caracteres mantienen ciertas reglas. Aunque las cadenas las veremos en extensión en el próximo capítulo, aquí veremos un avance sobre el tema:
 
 

#include <stdio.h>

main()
     {
      char saludo[16] = {'s', 'a', 'l', 'u', 'd', '\0', 'a', 'l', 'i', 'm', 'e', 'n', 't', 'o','\0' ,'s'};
      char saludo_compuesto[]= "Salud y alimentos";
      int i;

      printf("Un saludo corriente es %s\n", saludo);
      printf("Un saludo compuesto será: %s\n", saludo_compuesto);

      for (i=0; i < 16; i++)
            {
           printf("%c", saludo[i]);
            }
      printf("\n");

     }
 



Como novedades podemos observar que hemos situado en el array saludo[16] dos caracteres nulos '\0' y en el array saludo_compuesto[ ] no hemos especificado el tamaño del mismo. El programa se desarrolla definiendo dos arrays mostrandolos por pantalla mediante dos funciones printf() y a continuación muestra el array saludo[16] caracter por caracter mediante la función for().

Observa la curiosa salida del programa:
 

Un saludo corriente es salud
Un saludo compuesto será Salud y alimentos
saludalimentos



El mecanismo que usa C para leer una cadena de caracteres es leer los diferentes caracteres del array hasta que se encuentra un caracter nulo '\0' en la memoria. Como los arrays almacenan sus elementos de manera consecutiva, en el caso de que no hubieramos figurado el caracter nulo, el compilador seguiría leyendo carcteres por la memoria hasta encontrar el omitido carácter nulo. En nuestro caso el caracter nulo está situado en saludo[5] y en saludo[14]. De este razonamiento que la primera sentencia printf() de nuestro programilla sólo escriba por pantalla "salud".


En la definición de la varible saludo_compuesto[ ] hemos omitido el número de elementos. ¿por qué? Ocurre que cuando especificamos los elementos de un array durante la definición, podemos especificar el tamaño del array si queremos. Si no lo especificamos, el compilador "cuenta" los caracteres y reserva el espaco necesario para el alamacenamiento del array, añadiendo de forma automática el caracter '\0' para poder controlar el fin de la cadena.  En el caso de que no definamos los elementos del array debemos siempre especificar el tamaño del arrray. Observa como el array saludo_compuesto es leido entero. El caracter '\0', cuando lo leemos tiene una salida nula por pantalla.


Cuando leemos el array saludo[16] medinte el bucle for(), leerá todos los caracteres del array, incluso los nulos ('\0') aunque como son nulos nada saldrá por pantalla.

Una de las características fundamentaes de los arrays es su nombre  que  contiene la dirección del mismo. Como ya sabemos un poco acerca de los punteros, no es descabellado pensar que nos podemos referir a una array como si de un puntero se tratara. Veamos dos programas ilustrativos de este tema. primero veamos que ocurre cuando sumamos números enteros a direcciones de memoria:
 
 
 

#include <stdio.h>

main()
     {
      int t[]= {1, 2, 3, 4} /* No especificamos el tamaño */

      for(i=0; i< 4; i++)
           printf("t[%d] = %d    t = %d     t+%d = %d\n", i, t[i], t, i, t+i)); 
     }

 


Y el resultado de este programa es:



t[0] = 1    t = 743728     t + 0 = 743728 
t[1] = 2    t = 743728     t + 1 = 743732 
t[2] = 3    t = 743728     t + 2 = 743736 
t[3] = 4    t = 743728     t + 3 = 743740 



¡¡ Increible!! Cuando sumamos una unidad a t no le suma una unidad, suma 4 unidades. Recordemos que el nombre del array es la dirección del primer elemento del array, de manera que la siguiente equivalencia es muy cierta:
 

t == &t[0]



Como el array t[] es un array de enteros, lo que hace C cuando sumamos una unidad a una dirección de memoria perteneciente al array, lo que hacemos es sumarle una dirección de memoria más. El arrray cuestionado dispone de 4 elelmentos int y si recordais lo que ocupaba una variable int bajo UNIX, eran 4 bytes (A esta conclusión se llega asignanado el valor devuelto por sizeof(int) a una variable ). En nuestro caso podemos observar que t + 1 suma 4 unidades a al valor de t, de tal manera que cuando sumamos una unidad a una dirección de un array, el resultado nos está llevando al principio de la siguiente dirección del array. Si sumáramos dos unidades al valor de t, el resultado nos llevará dos direcciones más allá, pero sin embargo sumará 8 (2 * sizeof(int)) al valor de t. Esto es muy significativo, pues nos permitirá referirnos a un array como si de un puntero se tratara. Veamos:
