14.5.- Desarrollo de una librería y un fichero de cabecera propios 



Un veterinario desaseado pero simpático y usuario de Linux, tiene una hija merecedora y un hijo casadero y desea casar a ambos con un programador. Para ello ha establecido un concurso cuyas bases son el desarrollo de un programa que contenga una librería ".a", y un archivo propio de cabecera y que como premio ofrece la mano de alguno de sus dos hijos como pago de tan interesantes servicios. Veamos como desarrollar para el desaseado veterinario un programa con estos ingredientes. Puesto que la finalidad del programa no es relevante, vamos a desarrollar un programa que contenga alguna librería matemática y poca cosa más.

Vamos a usar tres funciones contenidas en tres archivos que se llamarán igual que las funciones a fin de identificarlos perfectamnete en nuestro directorio:

  • main.c ----->Contiene la función void main(void) o función principal

  • Mayusculas.c ----->Contiene la función Mayusculas(char *), que convierte a mayusculas el contenido de una cadena si fuera necesario

  • Minusculas.c ----->Contiene la función Minusculas(char *) que convierte a minusculas el contenido de una cadena si fuera necesario.

  • PerimetroRectangulo.c---->contiene la función PerimetroTriangulo (double, double) que devuelve el perimetro de un rectangulo usando la función sqrt()

Observa que en estos ficheros no hemos incluido el archivo <stdio.h>, pues no vamos a usar la entrada y salida de teclado y pantalla en el uso de estas funciones. El tercer fichero es PerimetroTriangulo.c que usa la función sqrt() de la librería matemática para hallar el perimetro de un rectangulo dados sus dos catetos:


/*Archivo PerimetroTriangulo.c**/

#include <math.h>


double PerimetroTriangulo(double C, double c)

{

    return (sqrt(C*C + c*c) + C + c);

}


Ya hemos desarrollado tres funciones en tres archivos '.c', ahora debemos compilarlas para obtener tres archivos .o con el siguiente comando:

gcc -c Mayusculas.c Minusculas.c PerimetroTriangulo.c


Que como ya hemos visto antes generará tres archivos llamados Mayusculas.o Minusculas.o PerimetroTriangulo.o.




El siguiente paso será generar una librería '*.a' que contenga las tres funciones anteriores y que puedan ser usadas en el futuro por otros programas: Como estas funciones apenas tienen que ver unas con otras salvo que pertenecen al mismo programa, vamos a llamar a la librería 'cajon_de_sastre' y que cuando se convierta en libreria '*.a' quedará como 'libcajon_de_sastre.a'. Esto lo haremos mediante el comando:

ar -r libcajon_de_sastre.a Mayusculas.o Minusculas.o PerimetroTriangulo.o


Y tendremos un archivo llamado libcajon_de_sastre.a. . Esta es la librería que podemos usar para una función main() determinada.

Para la generación de un índice para la biblioteca usaremos el comando ranlib de la siguiente forma:

ranlib libcajon_de_sastre.a


Y para ver el indice de la biblioteca indexada usartemos el comando nm:

nm -s libcajon_de_sastre.a

Probemos ahora como funciona nuestra librería desarrollando una función que use las tres funciones contenidas en ella:



/* Archivo main.c */

#include <stdio.h>


float PerimetroTriangulo(float C, float c);

void Mayusculas(char *);

void Minusculas(char *);


main()

{

     char cadena[80] = "La BaLa QuE DoBlO La EsQuInA";

     Minusculas(cadena);

     puts(cadena);

     Mayusculas(cadena);

             puts(cadena);

             printf("El perimetro de este triangulo es %g\n",      PerimetroTriangulo(10, 10));
}




Si suponemos que el archivo libcajon_de_sastre.a se encuentra en nuestro directorio de trabajo y que no es ni mucho menos el directorio donde usualmente se encuentran las librerias '.a'. el comado para compilar este programa es:

gcc main.c -L. -lcajon_de_sastre -lm -o ejucutable.exe


(*)

-L indica al compilador otro directorio donde encontrar librerias. Nosotros hemos situado un '.' que significa el directorio de trabajo, si la librería se encontrara en './lib/libcajon_de_sastre.a'(trayectoria relativa), invocariamos a gcc de la siguiente manera:

gcc main.c -L./lib -lcajon_de_sastre -lm -o ejecutable.exe

(*)

Si la librería se encontrara en /usr/resources/lib/ (trayectoria absoluta) , la compilación la hariamos así:

gcc main.c -L/usr/resources/lib/ -lcajon_de_sastre -lm -o ejecutable.exe


El parámetro -lcajon_de_sastre hace que el programa se linke o enlace con esta librería que contine las funciones necesarias para nuestro programa, además hemos de enlazarlo con la librería matemática mediante el parámetro -lm porque la función PerimetroTriangulo(), contenida en libcajon_de_sastre.a, usa la funcion sqrt() que a su vez está contenida en libm.a. Esta es la razón por la que hemos de incluir ambas librerias.



Aún así disponemos de otro problema. Nosotros como creadores de la librería sabemos lo que hemos hecho y no tenemos reparo en declarar nuestras funciones al principio de la librería. Esta labor que aparentemente es trivial puede convertirse en un calvario. Lo ideal es construir la librería y un archivo de cabecera a incluir con el fichero que va a usar la librería. Nosotros llamaremos a nuestro archivo 'cajon_de_sastre.h' y será algo parecido a esto:

/* Archivo cajon_de_sastre.h */
void Mayusculas(char *);
void Minusculas(char *);
double PerimetroTriangulo(double, double);


Este archivo contiene la declaración de las tres funciones contenidas en libcajon_de-sastre.a. El código de main.c sería con lo que suprimiriamos la declaración en el fichero de las funciones de la libreria:


/* Archivo main.c */

#include <stdio.h>

#include "cajon_de_sastre.h"

main()

{


     char cadena[80] = "La BaLa QuE DoBlO La EsQuInA";


     Minusculas(cadena);

     puts(cadena);

     Mayusculas(cadena);

     puts(cadena);

     printf("El perimetro de este triangulo es %g\n", PerimetroTriangulo(10, 10));
}


Las comillas dicen al compilador que debe de buscar el fichero cajon_de_sastre.h en el directorio de trabajo. Si estuviera en otro directorio, podemos usar trayectorias absolutas o relativas. Un ejemplo sería:

#include "/home/torpedo/include/cajon_de_sastre.h"
#include "./misincludes/fichero.h"


En todo caso será aconsejable disponer un fichero de texto con una descripción de las funciones de la librería, y una indicación de como compilar los programas que contengan dichas librerias, asi como la localización de los ficheros de cabecera necesarios.

El compilador gcc busca los archivos de cabecera en directorios creados para ese fin, usualmente en '/usr/include/'. Si en el momento de la compilación, queremos añadir directorios al ya existente y que el compilador disponga de otros directorios de busqueda de archivos de cabecera además de los predeterminados, , el compilador gcc permite hacerlo mediante la opción -I. Esta opción añade directorios de inclusión de archivos de cabecera a los ya habituales, usando la siguiente sinopsis:

gcc -I <Ruta>


Donde <Ruta> puede ser una trayectoria relativa o absoluta. Como ejemplos de trayectorias realtivas y absolutas puedes ver los siguientes ejemplos:

gcc -I./archivosH/proyecto1/ Archivo.c
gcc -I/home/torpedo/archivosH/proyecto1 Archivo.c
